Particionado de discos: fundamentos
▼

¿Qué es una partición?

Una partición es una división lógica de un disco duro o SSD. Es como dividir un armario grande en varios compartimentos: cada uno es independiente y puede tener su propia organización.

MBR vs GPT: dos formas de organizar particiones

Cuando un disco se inicializa hay que elegir un esquema de particionado. Hoy en día, siempre que la placa tenga UEFI, se usa GPT.

CaracterísticaMBRGPT
Año19832006
Máx. particiones4 primarias (o 3 + 1 extendida)128 (todas iguales)
Tamaño máx. disco2 TB9,4 ZB (prácticamente ilimitado)
FirmwareBIOS / UEFI legacySolo UEFI
RedundanciaUn único registro: si se daña, no arrancaCopia de seguridad al final del disco
Secure BootNoSí
Regla práctica

Si el equipo es de 2012 o posterior, usa GPT. Solo para equipos con BIOS antigua o SOs muy viejos se usa MBR.

Tipos de particiones (MBR)

TipoDescripciónMáximo
PrimariaPartición independiente, puede arrancar un SO4 por disco
Extendida«Contenedor» que alberga particiones lógicas1 por disco
LógicaDentro de la extendida, no arranca SO directamenteSin límite práctico
Ojo

En GPT no existen tipos primaria/extendida/lógica: todas las particiones son iguales y no hay límite de 4.

Sistemas de archivos

Es el «idioma» que usa el SO para leer y escribir datos en una partición.

SistemaSOMáx. archivoUso típico
NTFSWindows16 TBDisco principal de Windows, discos externos grandes
FAT32Todos4 GBUSBs y SD pequeñas, compatibilidad universal
exFATTodos16 EBUSBs grandes, sin límite de 4 GB
ext4Linux16 TBDisco principal de Linux
BtrfsLinux16 EBLinux moderno, snapshots, compresión
swapLinux—Memoria virtual (equivalente al pagefile)
Cuidado con FAT32

Si copias un archivo de más de 4 GB a un USB con FAT32, da error. Solución: formatear el USB como exFAT (universal) o NTFS (Windows).

Herramientas de particionado en Windows

  • Administrador de discos: gráfica, integrada (clic derecho en Inicio → Administración de discos).
  • diskpart: línea de comandos avanzada. Muy potente, pero no pide confirmación: un clean borra todo el disco.

Particionado y formateo en Linux
▼

Todo es un archivo

A diferencia de Windows (C:, D:...), Linux organiza todo en un árbol único que empieza en /:

/                     ← Raíz (equivalente a C:\)
├── home/             ← Carpetas personales
├── etc/              ← Configuración
├── var/              ← Logs, bases de datos
├── boot/             ← Arranque del kernel
├── dev/              ← Dispositivos (discos, USB)
├── mnt/              ← Montaje manual
└── media/            ← Montaje automático (USB, CD)

Los discos son archivos en /dev/

DispositivoSignificado
/dev/sdaPrimer disco SATA/USB
/dev/sda1Primera partición del primer disco
/dev/sdbSegundo disco
/dev/nvme0n1Primer SSD NVMe
/dev/nvme0n1p1Primera partición del NVMe

Montaje: conectar una partición al árbol

En Linux hay que «montar» cada partición en un directorio:

sudo mount /dev/sdb1 /mnt/datos   # Montar
ls /mnt/datos                      # Acceder
sudo umount /mnt/datos             # Desmontar

/etc/fstab: montaje automático

Para que una partición se monte sola al arrancar, se configura en /etc/fstab:

# <dispositivo>  <punto_montaje>  <fs>   <opciones>  <dump> <pass>
/dev/sda1        /                 ext4   defaults    0      1
/dev/sda2        /home             ext4   defaults    0      2
UUID=...         /mnt/datos        ext4   defaults    0      2
Mejor por UUID que por /dev/sdX

El nombre /dev/sdb puede cambiar si conectas los discos en otro orden. El UUID (que sacas con blkid) es único y estable.

Tipos de formateo

NivelQué haceCuándo
Bajo nivelReescribe la estructura físicaCasi nunca (puede dañar discos modernos)
Alto nivelCrea el sistema de archivosAl preparar una partición nueva
RápidoSolo borra la tabla de contenidosPara reutilizar rápido
CompletoBorra + verifica sector a sectorSi sospechas que hay sectores dañados

Herramientas en Linux

HerramientaTipoDescripción

Journaling: el diario del sistema de archivos

Los FS modernos (NTFS, ext4, Btrfs) llevan un registro de operaciones pendientes. Si hay un corte de luz, el SO consulta el diario y completa o deshace las operaciones incompletas, evitando pérdida de datos. FAT32 no tiene journaling: por eso hay que hacer «quitar hardware con seguridad» en los USBs.

RAID: discos que trabajan en equipo

RAID (Redundant Array of Independent Disks) combina varios discos para mejorar el rendimiento, la seguridad o ambos.

NivelDiscos mín.Cómo funcionaCapacidad útil
RAID 0
Stripping		2Datos repartidos entre discos: velocidad ×2. Si falla uno, pierdes todo.100%
RAID 1
Mirror		2Datos duplicados. Si falla un disco, el otro tiene la copia.50%
RAID 53Datos + paridad repartidos. Puede fallar 1 disco sin perder datos.(N-1) discos
RAID 104Espejo + stripping combinados: rápido y tolerante a fallos.50%

RAID por hardware vs software

TipoCómoCosteRendimiento
HardwareTarjeta controladora dedicada100-500 €Alto (la CPU no se ocupa)
SoftwareLo gestiona el SOGratuitoAlgo menor (usa CPU)
Para un taller o PYME

El RAID por software es suficiente y gratuito. Windows lo llama «volumen reflejado/seccionado» y Linux usa mdadm.

Volúmenes dinámicos en Windows

VolumenDescripciónEquivale a
SimplePartición normalPartición estándar
DistribuidoUne espacio de varios discosJBOD
SeccionadoReparte datos entre discosRAID 0
ReflejadoDuplica en dos discosRAID 1

LVM en Linux: volúmenes lógicos flexibles

LVM añade una capa entre los discos físicos y las particiones. Permite redimensionar en caliente sin reiniciar.

  • PV (Physical Volume): disco o partición real
  • VG (Volume Group): almacén que agrupa varios PV
  • LV (Logical Volume): partición virtual que sacas del VG, redimensionable

Clonación de discos y sistemas
▼

¿Qué es clonar un disco?

Es hacer una copia exacta bit a bit de un disco o partición en otro destino. El destino queda idéntico al origen: SO, programas, configuración y datos incluidos. Como una fotocopia perfecta.

¿Cuándo se clona?

SituaciónEjemplo
Migrar a disco nuevoCambiar HDD → SSD sin reinstalar Windows
Despliegue masivo30 PCs de un aula con la misma imagen
Copia de seguridad completaImagen del disco antes de una reparación
Sustitución por averíaRestaurar rápido un equipo averiado

Tipos de clonación

TipoQué copiaTamaño imagen
Disco a discoTodo el disco (incluida tabla de particiones)Igual al origen
Partición a particiónSolo una particiónSegún partición
Imagen de discoEl disco como archivo comprimidoMenor que el original
Imagen de particiónUna partición como archivoMenor aún

Herramientas gratuitas

HerramientaTipoInterfaz
ClonezillaLive USB/CDMenús de texto tipo asistente
RescuezillaLive USB/CDGráfica (basada en Clonezilla)
ddLinux CLILínea de comandos
Recomendada: Rescuezilla

Tiene interfaz gráfica y pasos guiados. Ideal para técnicos y alumnos que prefieren no pelearse con menús de texto.

dd: la navaja suiza de Linux

# Clonar disco sda a sdb
sudo dd if=/dev/sda of=/dev/sdb bs=4M status=progress

# Crear imagen de una partición
sudo dd if=/dev/sda1 of=/home/alumno/imagen.img bs=4M status=progress

# Restaurar imagen a una partición
sudo dd if=/home/alumno/imagen.img of=/dev/sdb1 bs=4M status=progress
ParámetroSignificado
if=Input file (origen)
of=Output file (destino)
bs=Block size (4M es óptimo)
status=progressMuestra el progreso
PELIGRO con dd

Si confundes if y of, destruyes el disco de origen. Verifica SIEMPRE con lsblk antes de ejecutar.

Copias de seguridad: fundamentos
▼

¿Por qué hacer copias de seguridad?

Los datos son lo más valioso del equipo. El hardware se reemplaza y el software se reinstala, pero los datos, si se pierden, se pierden.

Causas típicas de pérdida: fallo del disco, error humano (borrar por accidente), malware/ransomware, robo, incendio o inundación.

Tipos de copia

TipoQué copiaTamañoRestauración
CompletaTODOS los archivosGrandeRápida (un solo archivo)
IncrementalSolo lo cambiado desde la ÚLTIMA copia (del tipo que sea)Muy pequeñaLenta (completa + todas las incrementales)
DiferencialTodo lo cambiado desde la ÚLTIMA COPIA COMPLETAMediaMedia (completa + última diferencial)
Ejemplo: completa semanal + diaria
Lunes (completa):        [████████████████] 10 GB
Martes (incremental):    [██]                500 MB
Miércoles (incremental): [█]                 200 MB
Jueves (diferencial):    [███]               700 MB

Restaurar al miércoles con incrementales: L + M + X
Restaurar al jueves con diferencial:      L + J

La regla 3-2-1

La estrategia profesional:

  • 3 copias de los datos (el original + 2 copias)
  • 2 soportes diferentes (disco externo + nube, p. ej.)
  • 1 copia fuera del sitio (otro edificio, nube...)

RPO y RTO: métricas clave

MétricaSignificadoEjemplo
RPO
Recovery Point Objective		¿Cuántos datos puedes permitirte perder?RPO 1h = copias cada hora
RTO
Recovery Time Objective		¿Cuánto tiempo puedes estar sin el sistema?RTO 4h = restaurar en <4h

Herramientas integradas en Windows

HerramientaQué hace
Historial de archivosCopia automática de carpetas personales (Documentos, Escritorio...)
Imagen del sistemaCopia completa del disco (SO + programas + datos)
Puntos de restauraciónEstado del sistema (registro, drivers). NO los archivos personales
Historial de archivos

Microsoft lo marca como función heredada, pero sigue funcionando en Windows 10/11. Es la opción más sencilla para respaldar archivos personales de forma automática.

Copias de seguridad: automatización
▼
Automatizar = no depender de la memoria humana

La copia más peligrosa es la que «ibas a hacer mañana». Un sistema profesional se ejecuta solo, sin tu intervención.

Herramientas en Linux

HerramientaTipoDescripción
tarCLIEmpaqueta archivos en un .tar(.gz)
rsyncCLISincroniza dos ubicaciones. Solo copia lo cambiado
cronCLIProgramador de tareas del sistema
DuplicatiGráficaMultiplataforma, web UI, soporta nube (Drive, OneDrive, S3...)

tar: empaquetar y comprimir

# Crear archivo tar.gz
tar -czvf backup.tar.gz /home/alumno/Documentos/

# Ver contenido sin extraer
tar -tzvf backup.tar.gz

# Extraer
tar -xzvf backup.tar.gz -C /mnt/restauracion/
OpciónSignificado
-cCrear
-xExtraer
-zComprimir con gzip
-vVerbose (mostrar progreso)
-fNombre del archivo
-tListar contenido

rsync: sincronización inteligente

# Sincronizar a disco externo
rsync -avh --delete /home/alumno/Documentos/ /mnt/backup/Documentos/

# A otro equipo por SSH (cifrado)
rsync -avh -e ssh /home/alumno/Documentos/ usuario@192.168.1.100:/backup/
Por qué rsync es la estrella

Solo transfiere los bytes que han cambiado. La primera ejecución puede tardar, pero las siguientes son rapidísimas.

OpciónSignificado
-aArchive: preserva permisos, fechas, enlaces
-vVerbose
-hHuman-readable
--deleteBorra del destino lo que ya no existe en el origen
-e sshUsa SSH cifrado

cron: programar tareas

# Editar el crontab del usuario
crontab -e

# Formato:
# min hora día_mes mes día_sem  comando
#  0-59 0-23 1-31  1-12   0-7

# Backup diario a las 2:00 AM
0 2 * * * rsync -avh /home/alumno/Documentos/ /mnt/backup/

# Solo viernes a las 23:00
0 23 * * 5 tar -czf /mnt/backup/semanal_$(date +\%Y\%m\%d).tar.gz /home/alumno/

# Cada 6 horas
0 */6 * * * /home/alumno/scripts/backup.sh

Duplicati: backup gráfico con soporte para nube

Herramienta gratuita y de código abierto con interfaz web. Ofrece:

  • Backup incremental, cifrado y comprimido
  • Destinos: local, NAS, Google Drive, OneDrive, Dropbox, SFTP, S3...
  • Programación automática integrada
  • Verificación de integridad de las copias

Los datos no viajan «tal cual»: se organizan en capas, como un paquete postal.

PrivadaPública
Quién la veSolo tu red localTodo Internet
Quién la asignaTu router (DHCP)Tu ISP

DHCP y DNS: los dos invisibles

  • DHCP: cuando te conectas, el router te asigna IP, máscara, gateway y DNS automáticamente. Sin DHCP, lo configurarías a mano en cada dispositivo.
  • DNS: la «guía telefónica» de Internet. Cuando escribes www.google.es, tu equipo pregunta al DNS por la IP y luego se conecta a ella.
Tú escribes: www.google.es
    ↓
Tu PC → DNS: «¿qué IP tiene www.google.es?»
    ↓
DNS responde: «142.250.185.3»
    ↓
Tu PC se conecta a 142.250.185.3

Firewall: el portero de la red

Filtra el tráfico según reglas: permitir o bloquear por puerto, IP o programa.

  • Windows: Firewall de Windows Defender (integrado)
  • Linux: iptables o ufw (interfaz simplificada)

Ética en escaneos

Solo escanea equipos que sean tuyos o tengas permiso para escanear. Escanear redes ajenas puede ser ilegal.

Diagnóstico por capas

Cuando algo no va, sigue este orden para aislar el problema:

  1. Pila TCP/IP local: ping 127.0.0.1
  2. Puerta de enlace: ping [gateway]
  3. Internet sin DNS: ping 8.8.8.8
  4. Internet con DNS: ping www.google.es

El primer paso que falla te dice dónde está el problema (tarjeta, router, ISP o DNS).

Asistencia remota: herramientas profesionales
▼

¿Qué es la asistencia remota?

Permite controlar un equipo a distancia como si estuvieras sentado frente a él. Es fundamental en soporte técnico: ahorra desplazamientos, reduce tiempos y permite atender a clientes en cualquier parte del mundo.

Protocolos de acceso remoto

ProtocoloPuertoCifradoUso
RDP3389SíEscritorio remoto nativo de Windows
VNC5900No por defectoControl remoto multiplataforma open source
SSH22SíTerminal remota cifrada (sin escritorio)

Herramientas profesionales (2026)

HerramientaGratuitaVentajas
Asistencia rápida
Quick Assist		Sí (integrada)No requiere instalar nada. Integrada en Windows 10/11.
TeamViewerSí (personal)La más conocida. Funciona detrás de firewalls sin configurar.
AnyDeskSí (personal)Muy ligera y rápida. Buena en conexiones lentas.
RustDeskSí (open source)100% abierta. Puedes montar tu propio servidor.
Chrome Remote DesktopSíDesde el navegador, sin instalar software.
¿Cuál usar en cada caso?
  • Quick Assist: soporte entre equipos Windows en empresa
  • TeamViewer / AnyDesk: soporte a clientes remotos (la opción más habitual en talleres)
  • RustDesk: cuando quieres privacidad total y servidor propio
  • Chrome Remote Desktop: acceso puntual rápido sin instalar nada
  • SSH: administración de servidores Linux sin escritorio

Cómo funciona (simplificado)

La mayoría usan un servidor intermediario en la nube para que funcione aunque ambos estén tras routers/firewalls (NAT traversal).

SMB/CIFS: el protocolo de Windows

SMB (Server Message Block) es el protocolo que usa Windows para compartir carpetas e impresoras. También conocido como CIFS.

Samba: SMB en Linux

Samba es la implementación de SMB para Linux. Permite que un equipo Linux comparta carpetas y sea visible en una red de Windows (y viceversa).

sudo apt install samba

Permisos de compartición vs permisos NTFS

NivelSe aplica aEjemplo
Permisos de comparticiónAcceso por red (SMB)«Grupo Alumnos puede leer»
Permisos NTFSAcceso local y por red«Solo admin puede modificar»
Regla clave

Windows aplica el permiso más restrictivo de los dos. Si la compartición dice «lectura/escritura» pero NTFS dice «solo lectura», el resultado es solo lectura.

Mapeo de unidades de red

Asignas una letra (p. ej. Z:) a una carpeta compartida en otro equipo. Aparece en el Explorador como si fuera un disco local.

net use Z: \\192.168.1.10\Compartida /persistent:yes

# Para desconectar:
net use Z: /delete

Descubrir recursos en la red

# Desde Windows
net view \\192.168.1.10
net view /all

# Desde Linux
smbclient -L //192.168.1.10 -N

# Con Nmap
nmap --script smb-enum-shares -p 445 192.168.1.10
