Practica 27: Usuarios y grupos en Windows y Linux

Modulo: MF0219_2 - Instalacion y Configuracion de Sistemas Operativos
Unidad Formativa: UF0853 - Explotacion de las funcionalidades del sistema
Sesion: S7 - Usuarios y grupos
Duracion estimada: 4 horas (1 sesion)
Modalidad: Trabajo individual
Herramientas: VM Windows 10 + VM Ubuntu
Nota: Trabajaras de forma alternada en Windows y Ubuntu. Aunque se ven distintos, los conceptos son los mismos: usuarios, grupos, contrasenas, permisos asociados a la cuenta.

Objetivos de la practica

  • Crear, modificar y eliminar usuarios locales en Windows.
  • Diferenciar entre cuentas de Administrador y cuentas Estandar.
  • Crear usuarios en Linux desde la terminal con adduser.
  • Anadir un usuario a un grupo con usermod -aG.
  • Comprender el sentido de los grupos para administrar permisos.
  • Comprobar el funcionamiento de sudo.

1.2 - Crear un usuario estandar

  1. En la VM de Windows 10, abre Configuracion → Cuentas → Familia y otros usuarios.
  2. Pulsa Agregar otro usuario en este equipo.
  3. Cuando te pida correo, pulsa No tengo los datos de inicio de sesion de esta persona.
  4. Despues, pulsa Agregar un usuario sin cuenta de Microsoft.
  5. Crea el usuario alumno_test con contrasena.

1.3 - Promocionar a administrador

  1. Selecciona el usuario alumno_test → Cambiar tipo de cuenta.
  2. Cambialo a Administrador y acepta.
  3. Cierra sesion y entra como alumno_test. Comprueba que ahora puede instalar programas (basta con que ejecute el instalador y no le pida elevacion).
  4. Vuelve a tu sesion principal y devuelvelo a Estandar.

1.5 - Comandos en CMD (admin)

  1. Abre CMD como administrador.
  2. Lista usuarios: net user. Pega la salida.
  3. Lista grupos: net localgroup.
  4. Crea un usuario: net user prueba_cmd Pass1234! /add.
  5. Anadelo al grupo Usuarios: net localgroup Usuarios prueba_cmd /add.
  6. Borralo despues con: net user prueba_cmd /delete.

Parte 2 - Usuarios y grupos en Linux (75 min)

2.1 - Identificarte y ver tus grupos

  1. Abre la terminal en Ubuntu.
  2. Ejecuta y anota la salida:
    • whoami → tu nombre de usuario
    • id → UID, GID y grupos
    • groups → grupos a los que perteneces

2.2 - Archivos clave del sistema

  1. cat /etc/passwd | head -20
  2. cat /etc/group | head -20
  3. (Solo root) sudo cat /etc/shadow | head -5
ArchivoQue contiene
/etc/passwd
/etc/group
/etc/shadow

2.3 - Crear un usuario

  1. Crea el usuario: sudo adduser tecnico
  2. Sigue el asistente: pon contrasena, deja vacios los campos opcionales.
  3. Comprueba que se ha creado: id tecnico y ls /home.

2.4 - Crear un grupo y anadir el usuario

  1. Crea un grupo: sudo groupadd soporte.
  2. Anade tecnico al grupo: sudo usermod -aG soporte tecnico.
  3. Comprueba: groups tecnico → debe aparecer "soporte".
  4. Anade tambien tu propio usuario al grupo: sudo usermod -aG soporte $(whoami). (Hara falta cerrar sesion para refrescar grupos.)

2.5 - Cambiar contrasena

  1. Cambia la contrasena de tecnico: sudo passwd tecnico.
  2. Inicia sesion como tecnico: su - tecnico (introduce la nueva contrasena).
  3. Comprueba que estas como tecnico: whoami.
  4. Vuelve a tu usuario: exit.

2.6 - sudo: el delegado del root

En Ubuntu, el grupo sudo contiene a los usuarios con privilegios de administracion. Vamos a comprobarlo:

  1. su - tecnico → entra como tecnico.
  2. Intenta: sudo apt update. Te dira que no estas en sudoers.
  3. Vuelve a tu usuario: exit.
  4. Concedele permisos sudo: sudo usermod -aG sudo tecnico.
  5. Vuelve a entrar como tecnico (su - tecnico) y reintenta sudo apt update.

¿Que diferencia hay entre el usuario root y un usuario en el grupo sudo?

2.7 - Eliminar usuarios y grupos

  1. Borra al usuario tecnico (sin su carpeta home): sudo deluser tecnico.
  2. Si quieres borrar tambien la carpeta /home/tecnico: sudo deluser --remove-home tecnico.
  3. Borra el grupo: sudo groupdel soporte.
  4. Comprueba que ya no existe: id tecnico → "no such user".
