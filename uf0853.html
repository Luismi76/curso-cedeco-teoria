Gestión de archivos en Windows
▼
4 h Explorador · rutas · atributos · búsqueda

El Explorador de archivos

Es la herramienta gráfica con la que navegas por carpetas y archivos. Lo abres con Win+E o pinchando en la barra de tareas. La interfaz típica de Windows 10/11 tiene:

  • Panel izquierdo: Acceso rápido, OneDrive, Este equipo, Red.
  • Barra de direcciones: arriba; muestra la ruta actual y permite escribirla a mano.
  • Barra de búsqueda: a la derecha, busca dentro de la carpeta actual y subcarpetas.
  • Cinta de opciones: Inicio · Compartir · Ver. En Windows 11 es una barra más simple con iconos.

Concepto de ruta

Una ruta es la dirección completa de un archivo. Lee de izquierda a derecha:

Absoluta vs relativa

TipoDefiniciónEjemplo
AbsolutaEmpieza por la unidad o la raíz. No depende de dónde estés.C:\Users\Juan\foto.jpg
RelativaSe interpreta desde la carpeta actual...\foto.jpg (en la carpeta superior)

Las letras después del último punto le dicen al SO con qué programa abrir el archivo. Windows oculta las extensiones por defecto: actívalas en Explorador → Ver → Mostrar → Extensiones de nombre de archivo. Verlas es importante para detectar archivos sospechosos como factura.pdf.exe.

Atributos de archivo

Los atributos son etiquetas internas que controlan cómo el SO trata el archivo. Se ven en clic derecho → Propiedades.

AtributoLetraEfecto
Solo lecturaRNo se puede modificar (sí borrar). Protege de cambios accidentales.
OcultoHNo aparece en el Explorador a menos que actives «Mostrar ocultos».
SistemaSArchivo crítico del SO. No tocar.
Archivo (modificado)ALo usa el SO para saber si ha cambiado desde la última copia.

Desde CMD se ven y cambian con attrib: attrib +H secreto.txt oculta el archivo; attrib -H lo muestra de nuevo.

Operaciones básicas (CMD)

El gráfico es cómodo, pero un técnico debe conocer también la línea de comandos. Abre Símbolo del sistema (CMD) o PowerShell:

Búsqueda avanzada

En la barra de búsqueda del Explorador puedes usar filtros:

  • *.pdf — todos los PDF (comodín).
  • nombre:informe — archivos cuyo nombre contiene «informe».
  • tamaño:>10MB — más de 10 MB.
  • fechamodificación:esta semana — modificados esta semana.
  • tipo:imagen — solo fotos.

Para una búsqueda muy potente, hay que esperar a que el índice de Windows termine de mapear las carpetas (Panel de control → Opciones de indización).

Papelera de reciclaje

  • Borrar con Supr → va a la papelera, recuperable.
  • Borrar con Mayús+Supr → eliminación definitiva.
  • del en CMD no envía a la papelera.
  • La papelera tiene un tamaño máximo por unidad (configurable con clic derecho → Propiedades). Cuando se llena, los archivos más antiguos se borran de forma definitiva.
  • Archivos de red, USB o particiones externas no pasan por la papelera del PC: se borran directamente.
Para el examen

Diferencia absoluta (empieza por C:\) vs relativa (empieza por . o ..). Win+E abre el Explorador. attrib gestiona atributos desde CMD.

Gestión de archivos en Linux
▼
4 h Terminal · comandos · comodines · compresión

Por qué la terminal

En Linux hay gestores gráficos (Nautilus en GNOME, Dolphin en KDE), pero la terminal es lo que diferencia a un técnico: es más rápida, se puede automatizar y funciona también por SSH, sin escritorio. Abres una terminal con Ctrl+Alt+T en Ubuntu.

El prompt

Cuando abres una terminal ves algo como:

juan@portatil:~$
  • juan — usuario actual.
  • portatil — nombre del equipo (hostname).
  • ~ — directorio actual (la virgulilla equivale a /home/juan).
  • $ — prompt de usuario normal. Si es #, eres root.

Rutas en Linux

En Linux no hay letras de unidad. Todo cuelga de la raíz /:

/
├── home/        ← carpetas personales
│   └── juan/
├── etc/         ← configuración del sistema
├── var/         ← logs, colas, datos variables
├── tmp/         ← temporales (se borran al reiniciar)
├── usr/         ← programas instalados
└── root/        ← carpeta del superusuario

Atajos de ruta

SímboloSignificado
/Raíz del sistema
~Tu carpeta personal (/home/tú)
.Directorio actual
..Directorio superior
-(en cd) la carpeta anterior

Comodines (globbing)

  • * — cualquier cantidad de caracteres. ls *.txt = todos los .txt.
  • ? — un único carácter. ls foto?.jpg = foto1.jpg, foto2.jpg...
  • [abc] — cualquiera de esos caracteres.
  • {txt,pdf} — varias opciones. ls *.{txt,pdf}.

Redirección y tuberías

  • > — redirige la salida a un archivo (sobrescribe). ls > lista.txt.
  • >> — añade al final del archivo.
  • | — pasa la salida de un comando a otro. ls -la | grep ".txt".
  • 2> — redirige los errores. cmd 2> /dev/null los descarta.

Archivos ocultos

En Linux un archivo es oculto si su nombre empieza por punto. No es un atributo, es el propio nombre. Por eso existen .bashrc, .config/, .ssh/...

Para verlos: ls -a (todos) o ls -la (todos con detalles).

Compresión

FormatoCrearExtraer

tar.gztar -czf out.tar.gz carpeta/tar -xzf out.tar.gz
tar.bz2tar -cjf out.tar.bz2 carpeta/tar -xjf out.tar.bz2

Memotécnico para tar: create / xtract, verbose, file (siempre el último, antes del nombre), zgzip / jbzip2.

Editores de texto en consola

  • nano — fácil. nano archivo.txt. Ctrl+O guarda, Ctrl+X sale.
  • vim — potente pero curva de aprendizaje. Modos «normal» e «insertar».
Para el examen

En Linux un archivo es oculto si empieza por punto. ls -a los muestra. tar -czf crea un .tar.gz, tar -xzf lo extrae.

Equivalencia clave

C:\Users\<tú> en Windows ≈ /home/<tú> en Linux. Configuración del sistema: C:\Windows\System32\config (registro) ≈ /etc (texto plano).

Sistemas de archivos

El sistema de archivos es el «idioma» que usa el SO para escribir y leer datos en una partición.

SFSOTamaño máx. archivoUso típico
NTFSWindows16 TBDisco principal de Windows. Permisos, journaling, cifrado EFS.
FAT32Todos4 GBUSB pequeños, SD, ESP. Compatible universal.
exFATTodos16 EBUSB grandes y discos externos. Sin límite de 4 GB.

Trampa típica

«No puedo copiar una ISO de 5 GB al pendrive». Es FAT32, que limita a 4 GB por archivo. Solución: formatear el pendrive como exFAT.

Journaling

Es un «diario» donde el sistema apunta las operaciones antes de hacerlas. Si hay un corte de luz a media escritura, al reiniciar el SO mira el diario y deshace o completa la operación, evitando corrupción. Lo tienen NTFS, ext4, Btrfs. FAT32 no.

Letras de unidad en Windows

  • A:, B: — reservadas históricamente para disqueteras.
  • C: — disco principal donde está instalado el SO.
  • D:, E:... — otras particiones, DVD, USB.
  • Las letras se pueden cambiar desde Administración de discos (diskmgmt.msc) → clic derecho → Cambiar letra y rutas.

Montar unidades en Linux

En Linux, conectar un disco no le asigna letra. Hay que montarlo en una carpeta (punto de montaje):

# Ver discos conectados
lsblk

# Montar manualmente un pendrive
sudo mount /dev/sdb1 /media/pendrive

# Desmontar
sudo umount /media/pendrive

Ubuntu monta automáticamente los USB al insertarlos en /media/<usuario>/<etiqueta>. Para automontar al arranque se usa el archivo /etc/fstab.

El archivo /etc/fstab

Cada línea define una partición a montar: dispositivo, punto de montaje, sistema de archivos, opciones, dump y orden de fsck. Ejemplo:

UUID=abc-123  /         ext4    defaults  0 1
UUID=def-456  /home     ext4    defaults  0 2
UUID=xyz-789  none      swap    sw        0 0
Para el examen

FAT32 = límite de 4 GB por archivo. Para pendrives grandes con vídeos, usa exFAT. Configuración del sistema en Linux: /etc. Configuración del sistema en Windows: el registro (regedit).

Permisos, cuotas y cifrado
▼
4 h NTFS · rwx · chmod · BitLocker · LUKS

Por qué hay permisos

Un PC puede tener varios usuarios. Cada usuario tiene sus archivos y no debería poder ver ni modificar los de otro (ni tampoco los archivos del sistema). Los permisos son las reglas que el SO usa para decidir quién puede hacer qué con cada archivo.

Permisos NTFS (Windows)

Se asignan a usuarios o grupos. Se ven en clic derecho → Propiedades → Seguridad.

PermisoQué permite
LecturaVer el contenido y los atributos.
Lectura y ejecuciónLeer + ejecutar archivos ejecutables.
Mostrar contenido de carpetaSolo para carpetas: listar lo que hay dentro.
EscrituraModificar contenido y crear archivos nuevos.
ModificarLectura + escritura + borrar.
Control totalTodo, incluido cambiar los permisos y tomar posesión.

Herencia

Por defecto, una carpeta hija hereda los permisos de la carpeta padre. Si creas C:\Compartida\Equipo1, hereda los permisos de C:\Compartida. La herencia se puede romper en Propiedades → Seguridad → Avanzadas → Deshabilitar herencia.

Permisos efectivos y propietario

Cuando un usuario pertenece a varios grupos, sus permisos son la suma de los grupos, pero un denegar siempre gana a un permitir. Cada archivo tiene un propietario; el propietario siempre puede cambiar los permisos. Para tomar la propiedad: Seguridad → Avanzadas → Propietario → Cambiar.

Permisos en Linux

Mucho más simple que NTFS: cada archivo tiene 3 permisos para 3 niveles de usuario.

El primer carácter (tipo)

  • - archivo regular.
  • d directorio.
  • l enlace simbólico.
  • c, b dispositivos de carácter / bloque.

chmod (cambiar permisos)

Dos sintaxis equivalentes: octal (números) y simbólica (letras).

FormaEjemploResultado
Octalchmod 755 script.shrwxr-xr-x
Octalchmod 644 archivo.txtrw-r--r--
Octalchmod 600 .ssh/id_rsarw------- (solo el dueño)
Simbólicachmod u+x script.shañade ejecución al usuario
Simbólicachmod o-rwx privado/quita todo a «otros»
Simbólicachmod a+r doc.txtlectura para all (u+g+o)

Para carpetas, x significa «poder entrar» (con cd); sin x no se puede acceder aunque tengas lectura.

chown (cambiar propietario)

sudo chown juan:usuarios archivo.txt
sudo chown -R juan:juan /var/www/sitio/    # recursivo

Cuotas de disco

Son límites de espacio que el SO asigna por usuario o grupo. Útiles en aulas, oficinas y servidores con varios usuarios. Si un usuario alcanza la cuota, el sistema deja de aceptar escrituras suyas.

  • Windows: clic derecho en la unidad → Propiedades → Cuota → Habilitar administración. Se puede poner aviso (warn) y límite.
  • Linux: paquete quota. Comandos edquota (editar), quota -u usuario (ver), repquota (informe).

Cifrado de datos

Cifrar es codificar los datos para que solo se puedan leer con una clave. Si te roban el portátil, los datos siguen ilegibles.

HerramientaPlataformaQué cifraNota
BitLockerWindows Pro/EnterpriseUnidades completas (C:, pendrive...)Necesita TPM o contraseña al arranque.
EFSWindows ProArchivos/carpetas concretasVinculado a la cuenta del usuario.
LUKSLinuxParticiones enterasEstándar de facto en Ubuntu (opción «Cifrar la nueva instalación»).
VeraCryptMultiplataformaContenedores cifrados (archivos)Sucesor del difunto TrueCrypt.
FileVaultmacOSDisco completoEquivalente a BitLocker en Mac.

Clave de recuperación

Cuando activas BitLocker, Windows te ofrece una clave de recuperación de 48 dígitos. Es la única forma de recuperar los datos si olvidas la contraseña o el TPM falla. Guárdala en tu cuenta Microsoft, imprímela o anótala en un sitio seguro. Sin ella, los datos son irrecuperables.

Pregunta típica

«BitLocker sirve para...»: cifrar unidades completas y proteger los datos si te roban el portátil. No es un antivirus ni una copia de seguridad. EFS cifra archivos sueltos, LUKS cifra particiones en Linux.

Acciones sobre un dispositivo

  • Actualizar controlador: buscar uno más nuevo (en Windows Update o local).
  • Desinstalar dispositivo: elimina el driver. Si marcas «Eliminar software», borra los archivos; si no, queda en caché. Al reiniciar suele reinstalarse solo.
  • Deshabilitar: mantiene el driver pero desactiva el dispositivo. Útil para diagnóstico.
  • Propiedades → Controlador: ver versión, fecha y revertir a la versión anterior si la actualización ha roto algo.
  • Propiedades → Recursos: ver IRQ, DMA y direcciones de memoria que usa.
  • Propiedades → Eventos: historial de instalación y errores.

¿Qué es un driver?

Un controlador o driver es un programa que traduce las órdenes genéricas del SO al lenguaje específico del hardware. Sin driver, el SO ve el dispositivo pero no sabe «hablar» con él.

Plug and Play (PnP)

El sistema operativo detecta automáticamente el hardware nuevo al conectarlo y carga el driver adecuado si lo tiene. Casi todo lo USB es PnP. En equipos modernos los conflictos de IRQ son muy raros porque el sistema los gestiona él solo.

Drivers en Linux

  • La mayoría vienen en el kernel o como módulos. Por eso Linux suele detectar el hardware al instante.
  • Drivers privativos (Nvidia, algunas Wi-Fi) se gestionan desde Software y actualizaciones → Controladores adicionales.
  • Ver módulos cargados: lsmod. Cargar/descargar: modprobe / rmmod.
  • Ver hardware: lspci, lsusb, lshw.

Pantalla, energía y red
▼
4 h Resolución · planes de energía · IP/DNS/DHCP

Configuración de pantalla

Win+I → Sistema → Pantalla. Conceptos clave:

  • Resolución: número de píxeles. 1920×1080 = Full HD, 2560×1440 = QHD, 3840×2160 = 4K. Usa siempre la nativa del monitor (la que pone «recomendada»).
  • Escala: agranda letras e iconos sin bajar la resolución. 100 % por defecto, 150 % o 200 % en pantallas pequeñas con resolución alta.
  • Frecuencia de actualización: Hz. 60 Hz para uso normal, 120-144 Hz+ para juegos. Se cambia en Configuración avanzada de pantalla.
  • Orientación: Horizontal / Vertical / Invertida. Útil con monitores girados.
  • Modo nocturno / luz cálida: reduce el azul para descansar la vista.
  • HDR: mayor rango dinámico de color y brillo (necesita monitor compatible).

Múltiples pantallas

Win+P abre el selector rápido:

  • Solo pantalla del PC — apaga las externas.
  • Duplicar — la misma imagen en todas. Útil para presentaciones.
  • Extender — escritorio único repartido entre varias.
  • Solo segunda pantalla — apaga la del portátil.

IPv4 vs IPv6

  • IPv4: 4 números 0-255 separados por puntos. 4 mil millones de direcciones, ya agotadas a nivel global.
  • IPv6: 8 grupos hex separados por «:». Suficiente para muchas vidas. Cada vez más extendido.

IP pública vs privada

  • Privada: la que tienes en tu casa u oficina. Solo válida dentro de tu red. Rangos: 10.x.x.x, 172.16-31.x.x, 192.168.x.x.
  • Pública: la que tu operadora asigna al router. Visible desde internet. Una por router.
  • NAT hace la traducción entre tu IP privada y la pública del router.

Usuarios y grupos
▼
4 h Cuentas · UAC · sudo · directivas de contraseña

Por qué hay usuarios

Un SO multiusuario permite que varias personas usen el mismo equipo sin pisarse: cada uno tiene su carpeta, sus permisos y sus configuraciones. Y además permite separar privilegios: las cuentas normales no pueden romper el sistema porque no tienen acceso a archivos del SO.

Tipos de cuenta en Windows

CuentaPermisosUso
AdministradorTotal: instalar, cambiar config, gestionar otros usuarios.Solo para administración. No para uso diario.
EstándarUsar el sistema, modificar sus archivos. No puede tocar configuración global.Uso diario recomendado.
InvitadoMuy limitada. No guarda configuración.Acceso temporal. Deshabilitada por defecto en Win 10/11.
Cuenta MicrosoftOnline, vinculada a OneDrive, Office, sincronización entre dispositivos.Por defecto en instalaciones Home.
Cuenta localSolo en este PC, sin nube.Empresas, equipos compartidos, privacidad.

Crear un usuario en Windows

  1. Win+I → Cuentas → Otros usuarios → Agregar cuenta.
  2. «No tengo la información de inicio de sesión» → «Agregar un usuario sin cuenta Microsoft».
  3. Introduce nombre y contraseña.
  4. Para hacerlo administrador: selecciónalo → Cambiar tipo de cuenta → Administrador.

UAC (Control de cuentas de usuario)

Aunque seas administrador, Windows ejecuta los programas con permisos de usuario estándar por seguridad. Cuando un programa necesita privilegios elevados, sale el cuadro «¿Quieres permitir que esta app haga cambios?». Eso es UAC.

  • Si eres admin → confirmas con Sí.
  • Si eres usuario estándar → tienes que escribir la contraseña de un admin.
  • Nivel configurable en Panel de control → Cuentas → UAC.

Usuarios en Linux

  • root — el administrador. UID = 0. Lo puede todo. Por defecto su login está deshabilitado en Ubuntu; se usa sudo.
  • Usuarios normales — UID ≥ 1000. Solo tocan su carpeta /home/<usuario>.
  • Cuentas de sistema — UID 1-999. Las usa el SO para correr servicios (www-data, mysql, sshd). No se usan para login.

Archivos clave

ArchivoContenido
/etc/passwdUna línea por usuario: nombre, UID, GID, home, shell. Legible por todos.
/etc/shadowContraseñas cifradas (hash). Solo root puede leerlo.
/etc/groupDefinición de grupos y miembros.
/etc/sudoersQuién puede usar sudo y para qué. Se edita con visudo.

Ejemplo de línea en /etc/passwd:

juan:x:1000:1000:Juan Perez,,,:/home/juan:/bin/bash
 │   │   │    │    │            │           │
 │   │   │    │    └── GECOS    │           └── shell
 │   │   │    └── GID           └── carpeta home
 │   │   └── UID
 │   └── la x indica que la contraseña está en /etc/shadow
 └── nombre de usuario

Grupos

Un grupo es un conjunto de usuarios que comparten permisos. Más cómodo que dar permisos uno a uno. Ejemplos:

  • sudo (Ubuntu) — el grupo cuyos miembros pueden usar sudo.
  • video, audio — acceso a la cámara, sonido.
  • docker — usar Docker sin sudo.
  • En Windows: Administradores, Usuarios, Invitados, Usuarios del escritorio remoto...

Directivas de contraseña (Windows Pro)

En secpol.msc → Directivas de cuenta → Directiva de contraseñas puedes imponer:

  • Longitud mínima (recomendado ≥ 8 caracteres).
  • Complejidad activa: obliga mezclar mayúsculas, minúsculas, números y símbolos.
  • Vigencia máxima (caducidad cada N días).
  • Historial: no repetir las N últimas.
  • Bloqueo tras X intentos fallidos.

En Linux las directivas se configuran con PAM (/etc/pam.d/common-password) y /etc/login.defs.

Concepto clave

«Para instalar un programa en Windows, ¿qué cuenta hace falta?» → Administrador. En Linux: sudo (y pertenecer al grupo sudo).

Mantenimiento del sistema
▼
4 h Updates · liberar espacio · TRIM · tareas programadas

Mantenimiento preventivo

Tareas que un técnico realiza cada cierto tiempo (semanal, mensual) para que el equipo se mantenga rápido y seguro:

  1. Actualizar el sistema y las aplicaciones.
  2. Liberar espacio en disco.
  3. Comprobar discos y memoria.
  4. Optimizar arranque (desactivar programas innecesarios).
  5. Revisar antivirus y firewall.
  6. Verificar copias de seguridad.
  7. Limpiar polvo y comprobar temperaturas.

Actualizaciones

Windows Update

  • Configuración → Windows Update.
  • Tres tipos: de calidad (mensual, seguridad), de características (cada 12-24 meses, nueva versión), de drivers.
  • Horas activas para no reiniciar mientras trabajas.
  • Pausar hasta 5 semanas.
  • Equipos críticos: aplicar en horas no laborales y validar antes en un equipo piloto.

Ubuntu / Debian (apt)

sudo apt update             # refresca lista de paquetes
sudo apt upgrade            # instala actualizaciones disponibles
sudo apt full-upgrade       # también añade/quita paquetes si hace falta
sudo apt autoremove         # elimina paquetes huérfanos
sudo apt clean              # vacía la caché de .deb descargados

Las actualizaciones del kernel suelen necesitar reinicio; el resto, no.

Liberar espacio

Windows

  • Liberador de espacio en disco (clásico): cleanmgr. Marca todas las casillas, incluido Limpieza del sistema para borrar Windows.old y caché de actualizaciones.
  • Storage Sense (moderno): Win+I → Sistema → Almacenamiento. Automatiza la limpieza.
  • Configuración → Aplicaciones: ordenar por tamaño y desinstalar lo que no usas.
  • Carpetas a vigilar: %TEMP%, Descargas, papelera, caché del navegador, Windows.old.

Linux

  • sudo apt autoremove y sudo apt clean (ya vistos).
  • du -sh * para ver qué pesa en la carpeta actual.
  • ncdu / para una vista interactiva.
  • journalctl --vacuum-size=200M limita el tamaño de los logs.

Comprobación de discos

WindowsLinux
Errores lógicoschkdsk C: /Fsudo fsck /dev/sda1 (desmontada)
Sectores defectuososchkdsk C: /Rbadblocks /dev/sda
SMART (salud del disco)wmic diskdrive get model,status · CrystalDiskInfosudo smartctl -a /dev/sda

Desfragmentar vs TRIM

Es una de las preguntas más típicas. Aprende esto:

HDD (mecánico)SSD (estado sólido)
Cómo guardaPor sectores físicos en platos giratorios. Con el tiempo se fragmenta.En celdas NAND. No importa dónde estén físicamente.
Operación de mantenimientoDesfragmentar (reordena bloques contiguos)TRIM (avisa al SSD de qué bloques ya no se usan para que los borre)
FrecuenciaMensual / cuando supere el 10 %Automático en segundo plano
Herramienta WindowsOptimizar unidades (lo detecta automáticamente)Lo mismo: ejecuta TRIM si es SSD
Importante

Nunca desfragmentes un SSD con herramientas antiguas. Cada celda tiene un número finito de escrituras. Desfragmentar lo acortaría la vida sin ganar nada. Windows lo sabe y ejecuta TRIM en su lugar.

Tareas programadas

Windows: Programador de tareas

Lo abres con taskschd.msc. Permite ejecutar programas o scripts en horarios definidos.

  1. «Crear tarea básica» o «Crear tarea» (más opciones).
  2. Nombre → Desencadenador (diario, semanal, al inicio de sesión, al evento...).
  3. Acción (ejecutar programa). Indica ruta y argumentos.
  4. Condiciones (solo si está enchufado, si está inactivo...).
  5. Configuración (qué hacer si falla).

Linux: cron y systemd-timers

cron: ejecuta comandos en horarios definidos. Editas tu crontab con crontab -e:

# m h dom mon dow comando
0 22 * * 5  /home/juan/backup.sh    # Cada viernes a las 22:00
*/15 * * * * /usr/local/bin/check.sh # Cada 15 minutos
0 3 1 * *   /opt/scripts/mensual.sh  # El día 1 a las 3:00

systemd-timers: alternativa moderna, más potente y con mejor logging.

Limpieza física y temperaturas

  • Soplar el polvo cada 6-12 meses (aire comprimido, no aspiradora doméstica).
  • Cambiar pasta térmica de la CPU cada 3-5 años.
  • Revisar ventiladores y temperaturas con HWMonitor, Core Temp (Windows) o sensors (Linux).
  • CPU normal: 40-70 °C; bajo carga puntual hasta 85 °C es aceptable. Más de 90 °C sostenidos → problema térmico.
Para el examen

SSD → TRIM, no desfragmentar. HDD → desfragmentar. chkdsk en Windows / fsck en Linux comprueban errores del disco. sfc /scannow repara archivos del sistema corruptos.

Copias de seguridad y restauración
▼
4 h Tipos · 3-2-1 · Historial · Imagen · Duplicati · rsync

Por qué hacer copias

Los datos se pierden por muchos motivos: fallo de hardware, error humano (borrado accidental), robo, fuego, ransomware. La única forma fiable de protegerlos es tener copias.

Tipos de copia

TipoQué guardaVelocidadEspacioRestauración
Completa (full)Todo, cada vez.LentaMuchoUna sola copia: simple.
IncrementalSolo lo que cambió desde la última copia (de cualquier tipo).Muy rápidaPocoNecesitas la completa + todas las incrementales en cadena. Si pierdes una, te quedas a medias.
DiferencialLo que cambió desde la última completa.Media (crece)MedioSolo la completa + la última diferencial.
Sintética / Forever-incrementalSolo cambios, pero el software fusiona internamente.RápidaPocoComo una completa siempre.

Estrategia típica

Una completa semanal (domingo) + incrementales diarias (lunes a sábado). Cada semana se rota.

La regla 3-2-1

Memorízala así

3 copias de los datos → en 2 medios distintos → con 1 de ellas fuera de la ubicación principal (offsite o nube).

Ejemplo doméstico: original en el portátil, copia en un disco externo, copia en la nube. Si se incendia la casa, te queda la nube. Si una de las copias se corrompe, te quedan las otras dos.

Sincronización ≠ Copia de seguridad

OneDrive, Google Drive y Dropbox son sincronización, no backup. Si borras un archivo en el PC, se borra también en la nube. Una copia de seguridad real guarda versiones anteriores a las que puedes volver. La nube ayuda como una capa más, pero no sustituye al backup.

Herramientas en Windows

HerramientaQué hace
Historial de archivosCopia tus carpetas personales a un disco externo a intervalos. Recupera versiones anteriores. Configuración → Sistema → Almacenamiento → Avanzadas → Historial.
Copia de seguridad de Windows (Windows 7)Antigua, pero todavía disponible. Permite crear una imagen del sistema en disco externo. Panel de control → Copia de seguridad y restauración (Windows 7).
Puntos de restauraciónFoto de la configuración del sistema (registro, drivers, programas). Útil si una actualización rompe algo. Buscar «Crear un punto de restauración».
Restablecer este PCReinstala Windows. Opciones: conservar archivos o borrar todo.
OneDriveSincronización en la nube de tus carpetas. Hasta 5 GB gratis.

Punto de restauración vs Imagen del sistema

  • Punto de restauración: guarda solo el estado del sistema, no tus archivos personales. Útil si un driver malo o un programa rompió algo.
  • Imagen del sistema: copia exacta del disco (sistema + programas + archivos). Restaura todo el equipo a un estado anterior.

HerramientaPlataformaNotas
DuplicatiWin / Linux / MacSoftware libre. Copias cifradas, comprimidas, deduplicadas e incrementales a disco local, NAS o nube (S3, Google Drive, OneDrive, FTP, SSH...). Interfaz web. La hemos visto en la práctica 21.
rsyncLinux / macOSComando para sincronizar carpetas de forma muy eficiente (solo copia los cambios). Base de muchos scripts.

rsync en una línea

rsync -avh --delete /home/juan/  /mnt/backup/juan/
  • -a archive: copia recursiva preservando permisos, fechas, propietario.
  • -v verbose.
  • -h tamaños legibles.
  • --delete borra en el destino los archivos que ya no existen en origen (espejo).

Validación de las copias

Una copia que no se restaura no es una copia. Buenas prácticas:

  • Probar restauración al menos una vez al trimestre.
  • Verificar sumas/checksums.
  • Comprobar logs del backup tras cada ejecución.
  • Si es cifrado, no perder la contraseña (sin ella, las copias son inútiles).

Restauración bare-metal

«Bare-metal» = restaurar el sistema entero sobre un disco vacío. Pasos típicos:

  1. Arrancar desde el medio de recuperación del software (USB) o desde el USB de instalación de Windows → «Reparar el equipo».
  2. Seleccionar «Recuperación de imagen del sistema».
  3. Indicar la ubicación del backup.
  4. Restaurar y reiniciar.
Para el examen

3-2-1: 3 copias, 2 medios, 1 fuera. Incremental = desde la última copia (cualquiera). Diferencial = desde la última completa. Sincronizar en nube no es lo mismo que backup. Punto de restauración: solo sistema; imagen: todo.

★
Consejos para el examen final
▼
Antes del examen

Examen tipo test. 4 opciones, 1 correcta. Los fallos no restan. No dejes ninguna pregunta en blanco.

Las preguntas estrella

  1. Rutas y carpetas: C:\Users\<tú> en Windows ↔ /home/<tú> en Linux. Configuración del sistema en /etc. Comodín * y rutas relativas con ...
  2. Comandos básicos: ls -la (listar todo en Linux), dir (Windows). Punto al inicio = oculto en Linux. tar -czf crea, tar -xzf extrae.
  3. Sistemas de archivos: FAT32 → 4 GB por archivo. Para USB grandes → exFAT. NTFS para Windows, ext4 para Linux.
  4. Permisos Linux: rwx propietario / grupo / otros. chmod cambia permisos, chown cambia propietario. 755 = rwxr-xr-x.
  5. Cifrado: BitLocker = unidades completas (Windows). EFS = archivos sueltos. LUKS = particiones en Linux. La clave de recuperación es imprescindible.
  6. Cuentas Windows: Administrador (todo) vs Estándar (limitada). Para instalar programas hace falta administrador. UAC pide confirmación.
  7. Red: DHCP asigna IP automáticamente; DNS traduce nombres a IPs. 192.168.x.x es privada. ping prueba conectividad.
  8. Mantenimiento: SSD → TRIM (no desfragmentar). HDD → desfragmentar. chkdsk/fsck revisan discos. sfc /scannow repara archivos del sistema.
  9. Procesos: Ctrl+Mayús+Esc abre el Administrador de tareas. En Linux: top/htop, kill PID, kill -9 fuerza.
  10. Malware: Ransomware = cifra y pide rescate; Spyware = espía; Troyano = se disfraza; Gusano = se propaga solo por red; Rootkit = se oculta.
  11. Backup: regla 3-2-1 (3 copias, 2 medios, 1 fuera). Incremental = desde la última (cualquiera); diferencial = desde la última completa. Sincronizar en la nube no es lo mismo que backup.
  12. Accesibilidad: Narrador lee, Lupa amplía, Sticky Keys pulsa combinaciones una a una. Win+U abre la pantalla.

Estrategia durante el examen

  • Lee las 4 opciones antes de decidir: a veces hay dos plausibles y una es claramente la mejor.
  • Si dudas, descarta primero las imposibles. Quedarte entre dos te da 50 % en lugar de 25 %.
  • Marca y deja para el final las dudosas. Vuelve con calma.
  • Confía en la primera intuición salvo que recuerdes algo concreto que la contradice.
  • No dejes ninguna en blanco: como los fallos no restan, vale la pena adivinar.
