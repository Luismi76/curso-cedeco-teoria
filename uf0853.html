Gestión de archivos en Windows y Linux
▼

Conceptos básicos

  • Archivo: contenedor de datos con un nombre y una extensión.
  • Carpeta o directorio: agrupa archivos y otras carpetas.
  • Ruta: la dirección completa del archivo en el disco.
    • Windows: C:\Users\Juan\informe.docx
    • Linux: /home/juan/informe.txt
  • Extensión: las letras tras el último punto. Le dicen al SO con qué programa abrir el archivo (.docx, .jpg, .mp3...).

Archivos ocultos

  • Windows: atributo «Oculto» (clic derecho → Propiedades → Oculto).
  • Linux: el nombre empieza por punto (.bashrc, .config). Para verlos: ls -a.
Para el examen

«¿Qué hace que un archivo sea oculto en Linux?» → empezar por punto. «¿Cómo se listan los archivos ocultos?» → ls -a (o ls -la con detalles). Caen seguro.

Permisos, cuotas y cifrado
▼

Permisos NTFS (Windows)

PermisoQué permite
LecturaVer el contenido
EscrituraModificar contenido
Lectura y ejecuciónLeer y ejecutar archivos
ModificarLectura + escritura + borrado
Control totalTodo, incluido cambiar permisos

Se asignan a usuarios o grupos desde clic derecho → Propiedades → Seguridad.

Permisos en Linux

Cada archivo tiene 3 permisos para 3 niveles de usuario:

  • Comando para cambiar permisos: chmod 755 archivo o chmod u+x archivo.
  • Comando para cambiar propietario: chown usuario:grupo archivo.
  • El primer carácter indica el tipo: - archivo, d directorio, l enlace simbólico.

Cuotas de disco

Son límites de espacio que el SO asigna a cada usuario o grupo. Sirven para que un solo usuario no pueda ocupar todo el disco. Se gestionan en Windows desde Propiedades del disco → Cuota; en Linux con quota, edquota.

Cifrado de datos

HerramientaPlataformaPara qué
BitLockerWindows Pro/EnterpriseCifra unidades completas (C:, USB...)
EFSWindowsCifra archivos o carpetas concretas
LUKSLinuxCifra particiones enteras
VeraCryptMultiplataformaCifra contenedores de archivos
Pregunta típica

«BitLocker sirve para...»: cifrar unidades completas, para proteger los datos si te roban el portátil. No es un antivirus ni una copia de seguridad.

▼

Accesibilidad en Windows

HerramientaPara qué
NarradorLee en voz alta lo que aparece en pantalla.
LupaAmplía partes de la pantalla.
Alto contrasteCombinaciones de colores para baja visión.
Teclado en pantallaPara dificultades motoras o pantalla táctil.
Reconocimiento de vozControlar el PC por voz.
SubtítulosPara personas con problemas de audición.

Accesibilidad en Ubuntu / GNOME

Configuración → Accesibilidad. Incluye lector de pantalla (Orca), tecla pegajosa, lupa, alto contraste...

Drivers (controladores) de dispositivo

  • Un driver es un programa que permite al SO comunicarse con un dispositivo concreto (impresora, GPU, webcam...).
  • Sin driver, el SO no sabe «hablar» con el hardware.
  • Windows lo gestiona desde el Administrador de dispositivos (devmgmt.msc).
  • Linux suele incluirlos en el kernel o como módulos. Para drivers privativos (Nvidia): Software y actualizaciones → Controladores adicionales.

Plug and Play (PnP)

El PC detecta el dispositivo automáticamente al conectarlo y carga el driver adecuado si lo tiene. Casi todo lo USB es PnP.

Recuerda

El Administrador de dispositivos en Windows muestra todo el hardware del PC y avisa con un signo de admiración (⚠) si falta algún driver.

5
Pantalla, energía y red
▼

Pantalla

  • Resolución: número de píxeles (1920×1080 = Full HD, 3840×2160 = 4K).
  • Frecuencia de actualización: Hz. 60 Hz para uso normal, 144 Hz+ para juegos.
  • Escalado: aumenta el tamaño de letras e iconos sin bajar la resolución (útil en pantallas pequeñas con resolución alta).
  • Modo nocturno / luz cálida: reduce la luz azul para descansar la vista.

Energía

Plan / modoPara qué
EquilibradoLo normal. Rendimiento medio, ahorro medio.
Ahorro de energíaBaja brillo, frena CPU. Ideal con batería.
Alto rendimientoCPU al máximo, ventiladores al máximo.
SuspenderEl PC se duerme: gasta poco, despierta rápido.
HibernarGuarda la sesión en disco y se apaga del todo.

Red: lo básico

  • Dirección IP: identifica al equipo en la red. Ejemplo: 192.168.1.45.
  • Máscara de subred: divide la IP en parte de red y parte de host. Ejemplo: 255.255.255.0.
  • Puerta de enlace (gateway): el router. Por dónde sale el tráfico hacia internet.
  • DNS: traduce nombres (google.com) a IPs.
  • DHCP: protocolo por el que el router asigna IP dinámica automáticamente al conectarte.
  • MAC: identificador físico de la tarjeta de red. No cambia con la red.

IP estática vs dinámica

TipoCómo se asignaCuándo usarla
DinámicaAutomática (DHCP)Equipos normales en oficina o casa
EstáticaManualServidores, impresoras de red, NAS

Comandos útiles

AcciónWindowsLinux

Para el examen

DHCP = asignación automática de IP. DNS = traduce nombres a IPs. No los confundas.

Usuarios y grupos
▼

Tipos de cuenta en Windows

CuentaPermisos
AdministradorTotal: instala programas, cambia configuración del sistema, crea usuarios.
EstándarLimitado: usa el sistema, pero no puede instalar nada que afecte a todos los usuarios.
InvitadoMuy limitada (deshabilitada por defecto en Windows 10/11).
Cuenta MicrosoftCuenta online vinculada a OneDrive, Office, etc.
Cuenta localSolo en el equipo, sin nube.

Usuarios en Linux

  • root: el administrador. UID = 0. Lo puede todo.
  • Usuarios normales: UID ≥ 1000.
  • Cuentas de sistema: para servicios (no se usan para login).
  • Archivos clave: /etc/passwd (cuentas), /etc/shadow (contraseñas), /etc/group (grupos).

Comandos de Linux

ComandoPara qué
sudoEjecutar un comando con privilegios de root.
adduser nombreCrear un usuario.
passwd nombreCambiar la contraseña.
usermod -aG grupo usuarioAñadir un usuario a un grupo.
id / whoamiVer qué usuario eres y a qué grupos perteneces.
groupsListar los grupos del usuario actual.

Grupos

Un grupo es un conjunto de usuarios que comparten permisos. Es más fácil dar permisos a un grupo que a 50 usuarios uno por uno. Ejemplo en Linux: el grupo sudo permite usar sudo; el grupo video da acceso a la cámara.

Concepto clave

«Para instalar un programa en Windows, ¿qué cuenta hace falta?» → Administrador. Las cuentas estándar pueden usar el PC pero no pueden cambiar configuración que afecte a todos los usuarios.

Mantenimiento del sistema y procesos
▼

Mantenimiento preventivo (lo que se hace cada cierto tiempo)

  • Actualizar el sistema y las aplicaciones (Windows Update / apt upgrade).
  • Liberar espacio: archivos temporales, papelera, cachés del navegador.
    • Windows: Liberador de espacio o Storage Sense.
    • Linux: sudo apt autoremove, sudo apt clean.
  • Desfragmentar (HDD) en Windows. NO se desfragmentan los SSD: el SO ejecuta TRIM en su lugar.
  • Comprobar disco: Windows chkdsk; Linux fsck.
  • Revisar el antivirus y los logs.

Procesos: el Administrador de tareas y top

WindowsLinux
AbrirCtrl+Shift+Esc → Administrador de tareastop o htop en terminal
Ver procesosPestaña ProcesosListado en pantalla, se autoactualiza
Finalizar procesoBotón derecho → Finalizar tareakill PID o killall nombre
Ver uso de CPU/RAMPestaña Rendimientotop, free -h

Servicios y demonios

  • Servicio (Windows): proceso que se ejecuta en segundo plano (Windows Update, Spooler, etc.). Se gestiona con services.msc.
  • Demonio (Linux): lo mismo, pero suelen acabar en d (sshd, cupsd). Se gestionan con systemctl.

Visor de eventos / logs

  • Windows: Visor de eventos (eventvwr). Registra errores, avisos y actividad del sistema.
  • Linux: /var/log/ (syslog, auth.log...) o journalctl.

Malware: tipos típicos

TipoQué hace
VirusSe replica infectando archivos.
TroyanoSe hace pasar por un programa útil.
RansomwareCifra tus archivos y pide rescate.
SpywareEspía lo que haces (contraseñas, navegación).

Para el examen

Ransomware: cifra archivos y pide rescate (Wannacry, etc.). Spyware: espía. Troyano: se disfraza. No los confundas.

Copias de seguridad y restauración
▼

Tipos de copia de seguridad

TipoQué guardaVelocidadEspacio
CompletaTodo, cada vezLentaMucho
IncrementalSolo lo que cambió desde la última copia (de cualquier tipo)RápidaPoco
DiferencialLo que cambió desde la última completaMediaMedio

La regla 3-2-1

Memorízala así

3 copias de los datos → en 2 medios distintos → con 1 de ellas fuera de la ubicación principal (offsite o nube).

Por ejemplo: original en el portátil, copia en un disco externo, copia en la nube. Si se incendia tu casa, te queda la nube.

Herramientas de copia

HerramientaPlataformaNotas
Historial de archivosWindowsIncluida. Copia tus carpetas personales a un disco externo.
Copia de seguridad de WindowsWindowsIncluida. Imagen del sistema.
OneDrive / Google DriveMultiplat.Sincronización en la nube. No es lo mismo que backup, pero ayuda.
DuplicatiMultiplat.Software libre, copias cifradas, programadas.
rsyncLinuxComando para sincronizar carpetas.
Time MachinemacOSEl backup nativo de Mac.

Restauración de Windows

  • Punto de restauración: guarda la configuración del sistema (registro, drivers...). Útil si una actualización rompe algo.
  • Restablecer este PC: reinstala Windows conservando o no tus archivos.
  • Imagen del sistema: copia exacta del disco; restaura el sistema entero.
No las confundas

Sincronización en la nube (OneDrive) no es lo mismo que copia de seguridad. Si borras un archivo, la sincronización lo borra también. Una copia de seguridad real guarda versiones anteriores.

Consejos para el examen
▼
Antes del examen

Examen final. 20 preguntas tipo test, 4 opciones, 1 correcta. No restan los fallos. No dejes nada en blanco.

  1. Rutas y carpetas: C:\Users\<tú> en Windows ↔ /home/<tú> en Linux. Datos de configuración del sistema en /etc.
  2. Comandos básicos: ls -la (listar todo en Linux), dir (Windows). Punto al inicio = oculto en Linux.
  3. Permisos Linux: rwx propietario / grupo / otros. chmod cambia permisos, chown cambia propietario.
  4. BitLocker = cifrar unidades. EFS = cifrar archivos.
  5. Cuentas Windows: Administrador (todo) vs Estándar (limitada). Para instalar programas hace falta administrador.
  6. Red: DHCP asigna IP automática, DNS traduce nombres a IPs.
  7. Mantenimiento: SSD → no desfragmentar (TRIM); HDD → sí desfragmentar.
  8. Malware: ransomware = cifra y pide rescate. Spyware = espía. Troyano = se disfraza.
  9. Backup: regla 3-2-1 (3 copias, 2 medios, 1 fuera). Diferencia completa / incremental / diferencial.
