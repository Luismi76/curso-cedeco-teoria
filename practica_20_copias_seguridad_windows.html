Practica 20: Copias de Seguridad en Windows

Modulo: MF0219_2 — Instalacion y Configuracion de Sistemas Operativos
Unidad Formativa: UF0852 — Instalacion y actualizacion de sistemas operativos
Bloque: Clonacion y copias de seguridad
Duracion estimada: 5 horas (1 sesion)
Modalidad: Trabajo individual
Herramientas: Oracle VirtualBox + VM con Windows 10/11
Requisito importante: Para esta practica necesitas un segundo disco virtual en tu maquina virtual que servira como destino de las copias de seguridad. En VirtualBox, ve a Configuracion > Almacenamiento y anade un nuevo disco duro virtual (VDI) de al menos 20 GB al controlador SATA. Despues de arrancar Windows, inicializa y formatea el disco desde Administracion de discos (diskmgmt.msc).
Escenario: Trabajas como tecnico informatico en un despacho de abogados llamado Garcia & Asociados. El bufete maneja documentos legales criticos (contratos, expedientes judiciales, escrituras) y no tiene ningun sistema de copias de seguridad. Tras una reciente perdida parcial de datos por un fallo de disco, la direccion te ha encargado configurar una estrategia de backup completa utilizando las herramientas integradas de Windows. Debes implementar y documentar toda la solucion.

Objetivos de la practica

  • Comprender los tipos de copia de seguridad: completa (full), incremental y diferencial.
  • Configurar y utilizar el Historial de archivos de Windows para proteger documentos del usuario.
  • Crear una Imagen del sistema como copia de seguridad completa del disco del sistema operativo.
  • Crear y utilizar Puntos de restauracion para recuperar el sistema ante cambios no deseados.
  • Disenar un plan de copias de seguridad aplicando la regla 3-2-1.
  • Programar tareas de backup automatizadas con el Programador de tareas de Windows.

Parte 1 — Disenar un plan de backup (30 min)

Antes de tocar ninguna herramienta, un buen tecnico planifica. En esta parte vas a repasar los conceptos teoricos y disenar un plan de copias de seguridad para el despacho de abogados.

Tipos de copia de seguridad

Tipo Descripcion Ventajas / Inconvenientes
Completa (Full) Copia todos los datos seleccionados cada vez que se ejecuta, independientemente de si han cambiado o no. Restauracion rapida y sencilla. Requiere mucho espacio y tiempo en cada ejecucion.
Incremental Copia solo los datos que han cambiado desde la ultima copia (ya sea completa o incremental). Muy rapida y ocupa poco espacio. Restauracion mas lenta (necesita la completa + todas las incrementales).
Diferencial Copia todos los datos que han cambiado desde la ultima copia completa. Restauracion mas rapida que la incremental (solo necesita la completa + la ultima diferencial). Crece en tamano con el tiempo.

La regla 3-2-1

Regla 3-2-1: Es el estandar de la industria para proteger datos de forma fiable:
3 copias de los datos (el original + 2 copias de seguridad).
2 tipos de soporte diferentes (ej: disco duro interno + disco externo, o disco + nube).
1 copia fuera del sitio (offsite) — en otra ubicacion fisica o en la nube.

Conceptos clave: RPO y RTO

Concepto Definicion Ejemplo
RPO
(Recovery Point Objective)		 Cantidad maxima de datos que una organizacion puede permitirse perder, medida en tiempo. Si el RPO es de 1 hora, se deben hacer copias al menos cada hora.
RTO
(Recovery Time Objective)		 Tiempo maximo aceptable para restaurar el sistema y volver a estar operativo. Si el RTO es de 4 horas, se necesita un metodo de restauracion que no tarde mas de 4 horas.

Parte 2 — Historial de archivos (60 min)

El Historial de archivos es la herramienta integrada de Windows que realiza copias de seguridad automaticas de los archivos personales del usuario (Documentos, Escritorio, Imagenes, etc.). Funciona de forma similar a una copia incremental continua.

Paso 1: Preparar el segundo disco

  1. Si aun no lo has hecho, abre Administracion de discos (diskmgmt.msc).
  2. Verifica que el segundo disco esta En linea, inicializado (GPT) y formateado como NTFS.
  3. Asegurate de que tiene una letra de unidad asignada (ej: D: o E:).
  4. Anota la letra de unidad asignada al disco de backup.
Dato Valor obtenido
Letra de unidad del disco de backup
Tamano total del disco de backup
Sistema de archivos

Paso 2: Activar el Historial de archivos

  1. Abre Panel de control > Sistema y seguridad > Historial de archivos.
  2. Tambien puedes buscar "Historial de archivos" en el menu Inicio.
  3. Selecciona el segundo disco como unidad de destino.
  4. Haz clic en Activar.
  5. Ve a Configuracion avanzada y cambia la frecuencia de copia a Cada 10 minutos (para agilizar la practica).
  6. Verifica que las carpetas incluidas son las predeterminadas (Bibliotecas, Escritorio, Contactos, Favoritos).

Parte 3 — Imagen del sistema (50 min)

Una Imagen del sistema es una copia de seguridad completa (bit a bit) de las particiones del sistema operativo. Permite restaurar el equipo completo en caso de fallo grave del disco o del sistema.

Diferencia con la clonacion: Aunque ambos procesos crean una copia exacta del disco, la clonacion copia un disco a otro disco listo para usar directamente (disco de arranque). La imagen del sistema genera un archivo comprimido que necesita un proceso de restauracion para ser utilizado. La imagen es mas flexible para almacenamiento, mientras que el clon es mas rapido para recuperacion inmediata.
Sobre el nombre "(Windows 7)": En Windows 10/11 esta herramienta sigue llamandose "Copias de seguridad y restauracion (Windows 7)" porque Microsoft la mantiene como funcionalidad legacy (no la ha actualizado, pero tampoco la ha eliminado). Para copias de archivos personales, Microsoft recomienda Historial de archivos (Configuracion > Sistema > Almacenamiento > Opciones avanzadas) o, en Windows 11, Copia de seguridad de Windows. Sin embargo, para crear imagenes completas del sistema esta sigue siendo la opcion oficial integrada y funciona perfectamente.

Paso 1: Crear la imagen del sistema

  1. Abre Panel de control > Sistema y seguridad > Copias de seguridad y restauracion (Windows 7).
    Si no la encuentras: pulsa Inicio, escribe copias de seguridad y selecciona "Copias de seguridad y restauracion (Windows 7)". Tambien puedes ir directamente con control /name Microsoft.BackupAndRestoreCenter en CMD.
  2. En el panel izquierdo, haz clic en Crear una imagen de sistema.
  3. Selecciona En un disco duro y elige el segundo disco virtual.
  4. Verifica que la unidad del sistema (C:) esta seleccionada para incluirse en la imagen.
  5. Haz clic en Iniciar copia de seguridad.
  6. Espera a que finalice el proceso (puede tardar entre 10 y 30 minutos dependiendo del tamano).
  7. Cuando pregunte si deseas crear un disco de reparacion del sistema, selecciona No (en una VM no es necesario).

Parte 4 — Puntos de restauracion (40 min)

Los Puntos de restauracion capturan el estado del sistema operativo (archivos del sistema, registro, programas instalados) en un momento concreto. Permiten revertir cambios del sistema sin afectar los archivos personales del usuario.

Importante: Los puntos de restauracion no son copias de seguridad de tus archivos personales. Solo protegen los archivos del sistema, el registro de Windows y los programas instalados. Los documentos, fotos y archivos del usuario no se ven afectados por una restauracion del sistema.

Paso 1: Activar la proteccion del sistema

  1. Haz clic derecho en Este equipo (o Mi PC) y selecciona Propiedades.
  2. Haz clic en Proteccion del sistema (en el panel izquierdo o en Configuracion avanzada del sistema).
  3. Selecciona la unidad C: y haz clic en Configurar.
  4. Marca Activar proteccion del sistema.
  5. Ajusta el uso maximo de espacio en disco al 10%.
  6. Haz clic en Aceptar.
Dato Valor obtenido
Estado de la proteccion del sistema (C:)
Espacio maximo reservado

Paso 2: Crear un punto de restauracion manual

  1. En la misma ventana de Proteccion del sistema, haz clic en Crear...
  2. Escribe como descripcion: Antes de instalar software de prueba.
  3. Haz clic en Crear y espera a que se complete.
  4. Anota la hora exacta de creacion del punto.
Dato Valor obtenido
Descripcion del punto de restauracion

Paso 4: Restaurar al punto anterior

  1. Vuelve a Proteccion del sistema y haz clic en Restaurar sistema...
  2. Selecciona el punto de restauracion que creaste (Antes de instalar software de prueba).
  3. Haz clic en Buscar programas afectados para ver que se eliminara.
  4. Confirma y haz clic en Siguiente > Finalizar.
  5. El sistema se reiniciara automaticamente.
  6. Tras el reinicio, verifica:
    • El programa de prueba ya no esta instalado.
    • El archivo archivo_personal_prueba.txt del Escritorio sigue existiendo.

Parte 5 — Programar tareas con el Programador de tareas (30 min)

En un entorno profesional, las copias de seguridad deben ejecutarse de forma automatica sin intervencion del usuario. El Programador de tareas de Windows permite ejecutar scripts o programas de forma programada.

Paso 1: Crear un script de backup sencillo

  1. Abre el Bloc de notas.
  2. Escribe el siguiente script exactamente (adapta la letra de la unidad de backup si es diferente):
@echo off
REM Script de backup automatico - Bufete Garcia & Asociados
REM Fecha de creacion: Practica 20

set ORIGEN=%USERPROFILE%\Documents\Bufete_Garcia
set DESTINO=E:\Backups\Bufete_Garcia_%date:~6,4%%date:~3,2%%date:~0,2%
set LOG=E:\Backups\log_backup.txt

echo ============================================ >> "%LOG%"
echo Backup iniciado: %date% %time% >> "%LOG%"
echo Origen: %ORIGEN% >> "%LOG%"
echo Destino: %DESTINO% >> "%LOG%"
echo ============================================ >> "%LOG%"

if not exist "%DESTINO%" mkdir "%DESTINO%"

xcopy "%ORIGEN%" "%DESTINO%" /E /Y /I >> "%LOG%" 2>&1

echo Backup finalizado: %date% %time% >> "%LOG%"
echo. >> "%LOG%"

echo Backup completado correctamente.
pause
  1. Guarda el archivo como backup_bufete.bat en la carpeta Documentos.
  2. Ejecuta el script manualmente haciendo doble clic para verificar que funciona.
  3. Comprueba que se ha creado la carpeta de backup en el disco E: y que contiene los archivos.
Verificacion Resultado
El script se ejecuta sin errores?
Carpeta de backup creada en el disco de destino
Numero de archivos copiados

Paso 2: Programar la tarea automatica

  1. Abre el Programador de tareas (taskschd.msc o busca en el menu Inicio).
  2. En el panel derecho, haz clic en Crear tarea basica...
  3. Configura la tarea:
    • Nombre: Backup diario Bufete Garcia
    • Descripcion: Copia de seguridad automatica de los documentos del bufete
    • Desencadenador (Trigger): Diariamente
    • Hora de inicio: 13:00 (o la hora actual + 5 minutos para probar)
    • Accion: Iniciar un programa
    • Programa/script: Ruta completa al archivo backup_bufete.bat
  4. Marca la casilla "Abrir el dialogo Propiedades de esta tarea cuando haga clic en Finalizar".
  5. Haz clic en Finalizar.
  6. En las propiedades, marca Ejecutar tanto si el usuario ha iniciado sesion como si no (opcional, requiere contrasena).
