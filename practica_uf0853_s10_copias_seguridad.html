Práctica 10 · Copias de seguridad y restauración

UF: UF0853 · Sesión: 10 / 10 · Modalidad: individual
Entorno: VM Windows + VM Ubuntu · necesitas el disco virtual extra de la P3

De qué va. Última sesión. Vas a montar la estrategia 3-2-1 completa en un escenario realista: punto de restauración + historial de archivos + imagen del sistema en Windows, y Timeshift + rsync + Duplicati en Linux. Termina restaurando archivos «borrados a propósito» para probar que las copias funcionan. Una copia que nunca se ha probado, no es una copia.

Antes de empezar: necesitas el disco extra de la P3. Si lo eliminaste, créalo de nuevo y dale 4 GB (más espacio que en P3, vas a meter backups). En Windows, asegúrate de que E: existe y tiene espacio.

Antes de configurar nada, diseña tu estrategia para esta práctica:

2 Punto de restauración (Windows)

Recuerda: un punto de restauración guarda solo el estado del sistema (registro, drivers, programas), NO tus archivos personales. Útil para deshacer una actualización mala o un driver problemático.

Activar y crear
  1. Buscar «Crear un punto de restauración» (panel Propiedades del sistema).
  2. Selecciona la unidad C: → Configurar.
  3. Habilitar protección del sistema. Espacio máximo: 5 %.
  4. Aceptar. Ahora botón Crear: nombre Antes_practica_P10.
  5. Espera a que termine.
REM Desde PowerShell elevado
Enable-ComputerRestore -Drive "C:\"
Checkpoint-Computer -Description "Pre_P10_PS" -RestorePointType "MODIFY_SETTINGS"

REM Listar puntos existentes
Get-ComputerRestorePoint

REM Restaurar (NO ejecutar a menos que quieras de verdad)
REM Restore-Computer -RestorePoint 1
Pega (Ctrl+V), arrastra una imagen o haz clic para seleccionar

3 Historial de archivos (Windows)

Backup automatizado de tus carpetas personales a otra unidad (la E: o un disco externo). Útil para versiones anteriores de archivos.

  1. Buscar «Historial de archivos» (panel clásico). En Windows 11 está medio escondido.
  2. Seleccionar unidad → E: (la del P3).
  3. Activar.
  4. Configuración avanzada: frecuencia → cada 1 hora. Conservar versiones: 1 año.
  5. Configurar inclusiones: por defecto incluye Bibliotecas, Escritorio, Contactos y Favoritos.
  6. Excluir carpetas: añade Música (suele ser grande y prescindible).
Forzar una copia inicial
  1. «Ejecutar ahora» (botón).
  2. Espera unos minutos.
  3. Abre E: → carpeta FileHistory\<usuario>\ → verás los archivos respaldados con timestamps en el nombre.
Pega (Ctrl+V), arrastra una imagen o haz clic para seleccionar
Restauración (lo importante)
  1. Modifica C:\Users\<tú>\Documents\un_archivo_existente.txt: cambia su contenido y guarda.
  2. Espera unos minutos (o fuerza otra copia con «Ejecutar ahora»).
  3. Clic derecho en el archivo → Restaurar versiones anteriores.
  4. Verás un listado con timestamps. Selecciona una versión anterior.
  5. Restaura → confirma si quieres sustituir o conservar ambos.

4 Imagen del sistema (Windows)

Copia exacta de todo el disco. Si el disco muere, restauras la imagen en otro y arranca tal cual estaba. La función oficial sigue llamándose «Copia de seguridad y restauración (Windows 7)» aunque sigue funcionando en Win 10/11.

  1. Panel de control → Sistema y seguridad → Copia de seguridad y restauración (Windows 7).
  2. Panel izquierdo: Crear una imagen del sistema.
  3. Destino: En un disco duro → selecciona E:.
  4. Confirma qué unidades respaldar: C: (sistema). No añadas E: misma.
  5. Iniciar la copia de seguridad. Tarda 10-20 min en una VM pequeña.
Aviso de tamaño. Una imagen del sistema de una VM Windows ronda los 8-12 GB. Si tu E: es de 2 GB, no cabe. Sube E: a 20 GB redimensionando el VDI desde VirtualBox y la partición desde diskmgmt antes de continuar. O, alternativa, salta a la Sección 5 y haz solo Linux.

5 rsync · backup eficiente en Linux

rsync es la herramienta más usada de la historia de los backups en Linux. Solo copia los cambios, preserva permisos, comprime al vuelo y trabaja por red si quieres.

Backup local básico 
# Asegúrate de que /mnt/datos_linux (P3) está montado
df -h | grep datos_linux

# Backup espejo: sincroniza, borra en destino lo que ya no está en origen
sudo rsync -avh --delete ~/DatosCliente/ /mnt/datos_linux/backup_diario/

# Sin --delete: solo añade y actualiza, nunca borra
sudo rsync -avh ~/DatosCliente/ /mnt/datos_linux/backup_acumulativo/

ls -la /mnt/datos_linux/

Opciones que vas a usar siempre:

FlagSignifica
-aArchive: recursivo, preserva permisos, propietario, links, fechas
-vVerbose: muestra qué hace
-hHuman-readable: tamaños legibles
-zComprimir al vuelo (útil por red)
--deleteBorrar en destino lo que no está en origen (modo espejo)
--dry-runSimular sin hacer nada (PROBAR siempre antes de --delete)
-PProgreso + reanudable
--exclude=PATExcluir patrones
Probar la eficiencia incremental 
# Primera vez: copia todo (lento)
time sudo rsync -avh ~/DatosCliente/ /mnt/datos_linux/backup_diario/

# Segunda vez sin cambios: instantáneo
time sudo rsync -avh ~/DatosCliente/ /mnt/datos_linux/backup_diario/

# Cambia algo y vuelve a correr
echo "modificado" >> ~/DatosCliente/Documentos/contrato.txt
sudo rsync -avh --delete ~/DatosCliente/ /mnt/datos_linux/backup_diario/
# Solo copia el archivo modificado

Backup rotatorio diario (7 días)

cat <<'EOF' > ~/backup_diario.sh
#!/bin/bash
SRC="$HOME/DatosCliente/"
DEST="/mnt/datos_linux/diario_$(date +%w)"   # 0=domingo, 6=sábado
mkdir -p "$DEST"
rsync -avh --delete "$SRC" "$DEST/" >> ~/backup.log 2>&1
EOF
chmod +x ~/backup_diario.sh

# Programar en cron, cada noche a las 02:00
(crontab -l 2>/dev/null; echo "0 2 * * * /home/$USER/backup_diario.sh") | crontab -

# Probar manualmente
~/backup_diario.sh
ls /mnt/datos_linux/

Backup por SSH (¡la magia!)

# Backup desde tu VM Ubuntu a la otra VM (también Ubuntu o cualquier servidor SSH)
# Requiere ssh server en el destino
rsync -avhz -e ssh ~/DatosCliente/ usuario@10.10.10.20:/home/usuario/backup_remoto/

# Backup TIRANDO desde un servidor remoto a tu PC local
rsync -avhz usuario@10.10.10.20:/home/usuario/datos/ ~/copia_de_servidor/

6 Timeshift · snapshots del sistema (Linux)

Timeshift es a Linux lo que «Restaurar sistema» es a Windows: hace snapshots de archivos del SO (no de tus datos personales) que puedes restaurar.

sudo apt install -y timeshift
sudo timeshift --list
sudo timeshift --create --comments "Antes de la P10"

# Si te pregunta tipo: RSYNC (más sencillo y compatible)

sudo timeshift --list

O por GUI: busca «Timeshift» en el menú de aplicaciones. Es más cómodo configurar la frecuencia.

Probar restauración con cambio reversible 
# Cambiar el hostname del sistema
sudo hostnamectl set-hostname pruebatimeshift
cat /etc/hostname

# Restaurar al snapshot anterior
sudo timeshift --list
sudo timeshift --restore --snapshot '<ID-DEL-SNAPSHOT>' --yes
# Te avisa que va a reiniciar

# Tras reiniciar, comprobar
cat /etc/hostname    # vuelve al original

7 Duplicati · backup cifrado, incremental y multidestino

Duplicati es lo más cercano a una solución «de empresa» gratuita. Hace backup cifrado (AES-256), incremental, deduplicado y comprimido. Puede subir a S3, Google Drive, OneDrive, FTP, SSH, WebDAV, etc.

Si ya hiciste la práctica 21: Duplicati ya lo conoces. Esta sección es la versión multiplataforma y avanzada con varios backups.
Instalación (Linux) 
# Descargar el .deb más reciente desde duplicati.com
# O usar Mono
sudo apt install -y mono-runtime libmono-system-data4.0-cil
# Después: descarga el .deb e instala con sudo dpkg -i duplicati_*.deb
sudo apt --fix-broken install -y

# Arrancar la interfaz web
duplicati-server &

# Abrir en navegador
xdg-open http://localhost:8200
Crear el backup desde la UI web
  1. Add backup → Configure a new backup.
  2. Nombre: Backup_DatosCliente. Cifrado: AES-256. Frase: Duplicati_P10_2026! (apúntala, sin ella no podrás restaurar).
  3. Destination: Local folder → /mnt/datos_linux/duplicati/.
  4. Source data: ~/DatosCliente.
  5. Schedule: cada día a las 02:30.
  6. Options: retención «smart» (conserva diarios 7 días, semanales 4, mensuales 12).
  7. Save → Run now.
Pega (Ctrl+V), arrastra una imagen o haz clic para seleccionar
Restauración real
  1. Borra a propósito un archivo: rm ~/DatosCliente/Documentos/contrato.txt.
  2. En Duplicati: Restore → selecciona el backup → Next → elige fecha → marca el archivo → restaurar a la ubicación original.
  3. Verifica: cat ~/DatosCliente/Documentos/contrato.txt.
Por qué Duplicati es brutal en producción: con la misma config puedes apuntar a Backblaze B2, S3, Google Drive, OneDrive... La nube actúa de «1» en tu 3-2-1, cifrada de extremo a extremo (Duplicati cifra ANTES de subir, así que ni el proveedor puede ver los datos).

9 Síntesis · documenta tu política de backup

Como técnico, vas a entregar a tu cliente un documento con la política de backups que has configurado. Rellénalo.

RTO y RPO. Dos métricas que cualquier técnico debe saber:
  • RTO (Recovery Time Objective): cuánto puede estar el sistema caído. Si es 1h, necesitas restaurar en menos de 1h.
  • RPO (Recovery Point Objective): cuántos datos puedes permitirte perder. Si es 24h, basta con un backup diario. Si es 1h, necesitas hacer copia cada hora.
