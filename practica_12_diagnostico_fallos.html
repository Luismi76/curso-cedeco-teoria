Nota importante: En esta practica vas a trabajar como un tecnico real. No se te dara la solucion directamente — deberas pensar, investigar y razonar. Un buen tecnico no memoriza soluciones, sigue un proceso: Detectar sintomas > Analizar causas > Aplicar solucion > Verificar.

Objetivos de la practica

  • Identificar fallos comunes en sistemas Windows dentro de un entorno virtualizado.
  • Aplicar un proceso sistematico de diagnostico siguiendo una metodologia profesional.
  • Resolver problemas reales como lo haria un tecnico de soporte informatico.
  • Documentar el proceso completo de diagnostico y resolucion de forma clara y ordenada.

Contenido teorico — Metodologia de diagnostico

Las 4 fases del diagnostico sistematico

Todo tecnico profesional sigue un proceso ordenado para resolver problemas. No se trata de probar cosas al azar, sino de seguir un metodo:

Fase Descripcion Preguntas clave
1. Deteccion Observar los sintomas sin tocar nada Que ocurre? Cuando ocurre? Que mensaje aparece?
2. Analisis Plantear hipotesis sobre la causa Que componente puede estar fallando? Que ha cambiado recientemente?
3. Resolucion Aplicar la solucion mas probable Que herramienta necesito? Cual es la accion menos invasiva?
4. Verificacion Comprobar que el problema esta resuelto Funciona correctamente? El problema ha desaparecido? Hay efectos secundarios?

Herramientas de diagnostico en Windows

Estas son las herramientas principales que puedes utilizar para investigar y resolver problemas en Windows:

Herramienta Como acceder Para que sirve
Administrador de tareas Ctrl+Shift+Esc Ver procesos, rendimiento CPU/RAM/disco
Visor de eventos eventvwr.msc Consultar errores y advertencias del sistema
Configuracion del sistema msconfig Gestionar programas de inicio, arranque seguro
Simbolo del sistema cmd como administrador Ejecutar comandos de reparacion (sfc, chkdsk)
Modo seguro Shift+Reiniciar o msconfig Arrancar Windows con drivers minimos
Restaurar sistema rstrui.exe Volver a un punto de restauracion anterior
Administrador de discos diskmgmt.msc Ver particiones y estado de discos
Informacion del sistema msinfo32 Ver configuracion completa del hardware/software

PREPARACION DEL PROFESOR — Fallos a provocar en las VMs

Atencion: Esta seccion solo es visible en pantalla, no se imprime. Contiene las instrucciones para preparar las VMs de los alumnos con fallos intencionados. Prepara las VMs antes de la clase.

Fallo 1: Error de arranque (no bootable device)

Pasos para provocarlo:

  1. Apaga la VM del alumno completamente.
  2. Ve a VirtualBox > Configuracion > Almacenamiento.
  3. Selecciona el disco duro virtual que esta conectado al controlador SATA.
  4. Haz clic derecho y selecciona "Eliminar conexion" (esto desconecta el disco sin borrarlo).
  5. Alternativa: Ve a Configuracion > Sistema > Orden de arranque y desmarca "Disco duro", dejando solo "Optica" y "Red".

Sintoma esperado: La VM muestra "No bootable medium found" o "FATAL: No bootable medium found! System halted."

Solucion esperada: El alumno debe verificar la configuracion de almacenamiento en VirtualBox y reconectar el disco duro virtual, o corregir el orden de arranque en Configuracion > Sistema.

Fallo 2: Sistema extremadamente lento (programas en inicio)

Pasos para provocarlo:

  1. Dentro de Windows en la VM, abre el Explorador de archivos.
  2. En la barra de direcciones, escribe shell:startup y pulsa Enter.
  3. Crea un archivo llamado procesos_pesados.bat con el siguiente contenido:
    @echo off
    start /min cmd /c "for /L %%i in (1,1,999999999) do @echo %%i >nul"
    start /min cmd /c "for /L %%i in (1,1,999999999) do @echo %%i >nul"
    start /min cmd /c "for /L %%i in (1,1,999999999) do @echo %%i >nul"
    start /min cmd /c "for /L %%i in (1,1,999999999) do @echo %%i >nul"
  4. Tambien puedes usar msconfig para anadir programas pesados al inicio.
  5. Reinicia la VM para verificar que el efecto funciona.

Sintoma esperado: Windows tarda mucho en arrancar, el sistema va extremadamente lento, CPU al 100%.

Solucion esperada: El alumno debe abrir el Administrador de tareas (Ctrl+Shift+Esc) > identificar los procesos que consumen CPU > ir a la pestana de Inicio > deshabilitar los programas sospechosos. Tambien eliminar los archivos .bat de la carpeta shell:startup.

Fallo 3: Perfil de usuario danado / sin acceso

Pasos para provocarlo:

  1. Dentro de Windows, crea una cuenta de usuario secundaria (la que usara el alumno).
  2. Inicia sesion con esa cuenta para que se cree el perfil y luego cierra sesion.
  3. Inicia sesion con la cuenta de administrador.
  4. Navega a C:\Users\ y renombra la carpeta del perfil del usuario secundario (ej: de "Alumno" a "Alumno_BAK").
  5. Alternativa: Cambia la contrasena de la cuenta del alumno sin informarle.

Sintoma esperado: El alumno no puede iniciar sesion, aparece un perfil temporal, o se muestra un error de "No se puede cargar el perfil de usuario".

Solucion esperada: Entrar en Modo seguro > crear un nuevo usuario administrador o reparar el perfil desde el registro de Windows (regedit > HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList).

Fallo 4 (opcional): Driver de video corrupto

Pasos para provocarlo:

  1. Dentro de Windows, ve a Programas y caracteristicas.
  2. Desinstala las Guest Additions de VirtualBox.
  3. Alternativa: Ve al Administrador de dispositivos, localiza el adaptador de pantalla y desinstala el driver (marcando "Eliminar el software del controlador").
  4. Reinicia la VM.

Sintoma esperado: Resolucion fija de 800x600, sin aceleracion grafica, pantalla muy pequena.

Solucion esperada: Reinstalar las Guest Additions desde el menu Dispositivos de VirtualBox, o actualizar el driver desde el Administrador de dispositivos.

Fallo 5 (opcional): Servicio critico deshabilitado

Pasos para provocarlo:

  1. Dentro de Windows, abre services.msc (Ejecutar como administrador).
  2. Localiza el servicio "Cliente DHCP" (DHCP Client).
  3. Haz doble clic, cambia el tipo de inicio a "Deshabilitado" y haz clic en "Detener".
  4. Alternativa: Deshabilita el servicio "Windows Update" de la misma manera.
  5. Reinicia la VM.

Sintoma esperado: Sin conexion de red (si se desactivo DHCP) o sin capacidad de actualizaciones (si se desactivo Windows Update). El icono de red mostrara "Sin acceso a Internet".

Solucion esperada: Abrir services.msc > localizar el servicio deshabilitado > cambiar tipo de inicio a "Automatico" > hacer clic en "Iniciar".

Parte 6 — Preguntas de reflexion (30 min)

Responde a las siguientes preguntas con tus propias palabras. Cada respuesta debe tener un minimo de 3 lineas.

1. Describe tu proceso de diagnostico paso a paso. Que hiciste primero, segundo, tercero? Si volvieras a empezar, cambiarias algo?

2. Que herramienta de Windows te resulto mas util para diagnosticar? Por que?

3. Cual es la diferencia entre "Modo seguro" y un arranque normal? En que situaciones usarias el Modo seguro?

4. Si un cliente te describe por telefono que su ordenador "no enciende", que preguntas le harias para empezar a diagnosticar el problema? (minimo 5 preguntas)

5. Que diferencia hay entre un problema de hardware y un problema de software? Como los distinguirias?

6. Por que es importante verificar que el problema esta resuelto ANTES de devolver el equipo al cliente? Que podria pasar si no lo haces?
