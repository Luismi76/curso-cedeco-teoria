Practica 01: Identificacion de Componentes de un Equipo Informatico

Modulo: MF0953_2 - Montaje de Equipos Microinformaticos
Unidad Formativa: UF0861 - Montaje y Verificacion de Componentes
Bloque: 1 - Montaje simulado de equipos
Duracion estimada: 4 horas (1 sesion)
Modalidad: Trabajo individual con entrega obligatoria
Herramienta: PC Building Simulator (Steam) / Imagenes proporcionadas por el profesor
Nota: Esta practica se realiza de forma simulada utilizando PC Building Simulator o, alternativamente, imagenes y fichas tecnicas proporcionadas por el profesor. No se trabaja con hardware real.

Informacion sobre la practica

Modalidad de trabajo

Esta practica se realiza de forma individual. Cada alumno trabajara con su propia instancia del simulador (o con el conjunto de imagenes proporcionado) y entregara su propio informe.

Requisitos previos

Antes de comenzar, asegurate de que dispones de:

Requisito OK
PC Building Simulator instalado y funcionando (o imagenes proporcionadas por el profesor)
Acceso a Internet para consultar fichas tecnicas de componentes
Procesador de texto para redactar el informe (Word, LibreOffice Writer, Google Docs)
Herramienta de captura de pantalla (tecla Impr Pant, Recortes de Windows, o similar)

Objetivos

  • Reconocer visualmente los componentes principales de un ordenador de sobremesa.
  • Comprender la funcion basica de cada componente dentro del sistema.
  • Identificar las conexiones entre componentes y sus dependencias.
  • Detectar posibles incompatibilidades entre componentes.
  • Familiarizarse con la terminologia tecnica del hardware.

Contenido teorico previo

Antes de comenzar la actividad practica, repasa la siguiente tabla de componentes y sus funciones. Este conocimiento es imprescindible para completar la practica.

Componentes principales

Componente Funcion principal Analogia sencilla
CPU (procesador) Ejecuta las instrucciones y realiza los calculos del sistema El "cerebro" del ordenador
RAM (memoria) Almacena temporalmente los datos que la CPU necesita de forma inmediata La "mesa de trabajo" - cuanto mas grande, mas cosas puedes tener a mano
Disco duro / SSD Almacena datos de forma permanente (sistema operativo, programas, archivos) El "armario" - guarda todo aunque se apague el ordenador
Placa base (motherboard) Conecta todos los componentes entre si y permite la comunicacion entre ellos La "columna vertebral" del sistema
Fuente de alimentacion (PSU) Convierte la corriente alterna de la red en corriente continua y la distribuye El "corazon" que suministra energia a todos los componentes
Tarjeta grafica (GPU) Procesa la imagen y la envia al monitor El "artista" que dibuja todo lo que ves en pantalla
Disipador / ventilador CPU Evacua el calor generado por el procesador El "aire acondicionado" del procesador
Caja (chasis / torre) Aloja y protege todos los componentes; organiza el flujo de aire La "casa" donde vive todo el hardware

Conectores y buses importantes

Conector / Bus Conecta... Se encuentra en...
Socket CPU Procesador - Placa base Placa base (zona central-superior)
Ranuras DIMM RAM - Placa base Placa base (junto al socket)
PCIe x16 Tarjeta grafica - Placa base Placa base (zona inferior)
SATA datos Disco duro/SSD - Placa base Placa base (borde derecho)
SATA alimentacion Disco duro/SSD - Fuente Cables de la fuente
ATX 24 pines Fuente - Placa base (alimentacion principal) Placa base (borde derecho)
EPS 4+4 pines Fuente - Placa base (alimentacion CPU) Placa base (esquina superior izquierda)
PCIe 6+2 pines Fuente - Tarjeta grafica Cables de la fuente / lateral de la GPU
M.2 SSD NVMe - Placa base Placa base (entre PCIe y chipset)

Parte 4 - Mapa de conexiones (30 min)

Dibuja (a mano o con una herramienta digital) un esquema simplificado que muestre como se conectan los componentes entre si. No necesita ser artistico, pero debe reflejar:

  • Todos los componentes principales.
  • Las lineas de conexion entre ellos (datos y alimentacion).
  • Diferencia entre cables de datos y cables de alimentacion (usa colores o tipos de linea distintos).
Ejemplo de formato: La placa base en el centro, con lineas que salen hacia la CPU, RAM, GPU, disco, etc. Las lineas de la fuente van a la placa base, a los discos y a la GPU.

Parte 5 - Analisis y preguntas de comprension (60 min)

Responde a las siguientes preguntas de forma razonada. Cada respuesta debe tener un minimo de 3 lineas y demostrar comprension tecnica, no solo memorizacion.

Preguntas sobre fallos de componentes

1. Que pasaria si falla la memoria RAM? Describe los sintomas que podria presentar el equipo (al arrancar y durante el uso).

2. Que ocurre si la fuente de alimentacion no funciona? El equipo dara alguna senal de vida? Justifica tu respuesta.

3. Si el disco duro falla pero el resto de componentes estan bien, el equipo arrancara? Que veras en pantalla? Se pierden los datos?

4. Que sintomas tendria un equipo cuyo disipador de CPU se ha desprendido o cuyo ventilador de CPU ha dejado de funcionar?

Preguntas sobre compatibilidad

5. Se puede usar cualquier procesador en cualquier placa base? Explica que es el "socket" y por que es importante. Pon un ejemplo de un socket Intel y otro AMD actuales.

6. Se puede instalar un modulo de RAM DDR4 en una placa base que solo soporta DDR5? Por que? Que mecanismo fisico impide la instalacion incorrecta?

7. Como puedes saber si una tarjeta grafica es compatible con tu placa base? Y como sabes si tu fuente de alimentacion es suficiente para esa tarjeta grafica?

Preguntas de razonamiento

8. Ordena los siguientes componentes de MAS critico a MENOS critico para que el equipo pueda encender y mostrar imagen. Justifica el orden: Tarjeta grafica dedicada, RAM, Disco duro, CPU, Fuente de alimentacion.

9. Un amigo te dice que quiere mejorar su ordenador porque "va muy lento". Que preguntas le harias antes de recomendarle cambiar un componente? Nombra al menos 4 preguntas y explica que informacion te daria cada una.

10. Observa el equipo que has analizado. Si tuvieras que proponer UNA mejora para aumentar el rendimiento general, cual seria y por que? Justifica tu eleccion tecnicamente.
