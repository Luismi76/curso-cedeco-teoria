Arquitectura del sistema y firmware
▼

Las capas del sistema

Un sistema microinformático funciona en capas. Cada capa habla con la de arriba y la de abajo, pero no se salta ninguna. Esto es lo que tienes que saber:

  • Hardware: los componentes físicos (CPU, RAM, disco, placa base...).
  • Firmware (BIOS/UEFI): el primer programa que se ejecuta al encender. Está «grabado» en un chip de la placa base.
  • Sistema operativo: Windows, Linux, macOS. Hace de intermediario entre el usuario y el hardware.
  • Aplicaciones: los programas que tú abres (navegador, ofimática...).

Hardware base vs software base

  • Hardware base: lo mínimo para que el PC arranque (placa base, CPU, RAM, almacenamiento, fuente).
  • Software base: firmware + sistema operativo. Sin ellos, el hardware no hace nada.

BIOS vs UEFI

BIOS (antiguo)UEFI (moderno)
InterfazTexto, sin ratónGráfica, con ratón
ArranqueLento, limitadoRápido (Secure Boot, Fast Boot)
DiscosHasta 2 TB (MBR)Más de 2 TB (GPT)
SeguridadBásicaSecure Boot (impide arrancar SO no firmados)
Para el examen

UEFI es lo moderno: interfaz gráfica, arranque rápido, GPT (discos > 2 TB), Secure Boot. La BIOS tradicional es texto, MBR, lenta. Esto cae siempre.

Funciones del sistema operativo
▼

El SO es el «director de orquesta» del PC. Sus funciones principales:

FunciónQué hace
Gestión de procesosReparte el tiempo de CPU entre los programas abiertos. Decide qué se ejecuta y cuándo.
Gestión de memoriaAsigna RAM a cada programa y se asegura de que no se «pisen». Si hace falta, usa memoria virtual (disco).
Gestión de archivosOrganiza los datos del disco en carpetas y archivos. Usa un sistema de archivos (NTFS, ext4...).
Gestión de E/SComunica el software con los periféricos a través de los drivers.
Interfaz de usuarioTe da una forma de interactuar (escritorio gráfico o terminal).
SeguridadUsuarios, contraseñas, permisos: controla quién puede hacer qué.

4
Estructura del sistema operativo
▼

El kernel: el corazón del SO

El kernel (o «núcleo») es la parte más importante del SO. Está en contacto directo con el hardware y gestiona:

  • El tiempo de CPU (qué proceso se ejecuta en cada instante).
  • La memoria RAM (qué proceso usa qué zona).
  • El acceso a los dispositivos (disco, tarjeta de red, USB...).
  • La seguridad: separa los procesos para que no se interfieran.

Capas del SO (visión simplificada)

NivelDescripción
KernelNúcleo. Habla con el hardware.
Servicios del sistemaProcesos que arrancan con el SO (red, audio, impresión...).
Shell / interfazEl intérprete (CMD, Bash) o el escritorio gráfico.
AplicacionesLo que ejecuta el usuario.

Modos de ejecución

  • Modo kernel (privilegiado): acceso total al hardware. Solo el SO entra aquí.
  • Modo usuario: con restricciones. Es donde se ejecutan tus aplicaciones.

Esta separación evita que un programa pueda «cargarse» el sistema por error.

Gestión de memoria y procesos
▼

Procesos e hilos

  • Proceso: un programa en ejecución (Chrome abierto = un proceso).
  • Hilo (thread): una «línea de ejecución» dentro del proceso. Un proceso puede tener varios hilos a la vez.
  • Multitarea: el SO va alternando rapidísimo entre procesos para que parezca que todo va a la vez.

Estados de un proceso

EstadoSignificado
En ejecuciónEstá usando CPU ahora mismo.
ListoPuede ejecutarse, pero la CPU está ocupada con otro.
BloqueadoEsperando algo (datos del disco, una pulsación de teclado...).
TerminadoHa finalizado.

Memoria virtual y archivo de paginación

¿Qué es?

Cuando la RAM se llena, el SO usa una zona del disco como si fuera RAM extra. En Windows se llama archivo de paginación (pagefile.sys); en Linux es la partición swap. Es mucho más lenta que la RAM real.

Para el examen

Si te preguntan qué pasa cuando la RAM se llena: el SO usa memoria virtual / swap en el disco. Por eso un PC con poca RAM y disco mecánico va lentísimo.

Linux: distribuciones
▼

¿Qué es una distro?

Una distribución de Linux es un SO completo formado por:

  • El kernel Linux (común a casi todas).
  • Utilidades del sistema (GNU coreutils, etc.).
  • Un entorno de escritorio (GNOME, KDE, XFCE...).
  • Un gestor de paquetes (apt, dnf, pacman...) para instalar software.
  • Aplicaciones preinstaladas.

Familias y ejemplos

FamiliaDistros conocidasGestor de paquetes
DebianDebian, Ubuntu, Linux Mint, Kaliapt (.deb)
Red HatFedora, RHEL, CentOS, Rockydnf / yum (.rpm)
ArchArch, Manjaropacman
SUSEopenSUSE, SLESzypper (.rpm)

Ubuntu: la distro recomendada para empezar

  • Basada en Debian.
  • Versiones LTS (Long Term Support) cada 2 años, con soporte 5 años.
  • Entorno gráfico GNOME por defecto.
  • Gestor de paquetes: apt.
  • Variantes: Kubuntu (KDE), Xubuntu (XFCE), Ubuntu Server (sin escritorio).
Truco

Si te dicen «Debian»: piensa en apt y archivos .deb. Si te dicen «Red Hat / Fedora»: dnf y .rpm. Esa correspondencia distro-gestor es típica de examen.

Particiones y sistemas de archivos
▼

Esquemas de particionado: MBR vs GPT

MBRGPT
AntigüedadAntiguo (~1983)Moderno
Tamaño máximo de disco2 TB> 9 ZB (ilimitado en la práctica)
Particiones primariasMáx. 4Hasta 128 (en Windows)
Firmware compatibleBIOSUEFI
Resistencia1 sola tablaTabla principal + copia

Tipos de particiones (esquema MBR clásico)

  • Primaria: arrancable. Máx. 4 por disco.
  • Extendida: sustituye a una primaria para poder crear más particiones (lógicas) dentro.
  • Lógica: dentro de la extendida. Sin límite práctico.

Con GPT esta distinción ya no tiene sentido: todas son «iguales» y puedes tener muchísimas.

Sistemas de archivos

SFSO típicoComentarios
NTFSWindowsPor defecto en C:. Permisos, cifrado, compresión.
FAT32Pendrives, SDCompatible con todo. Archivo máx. 4 GB.
exFATPendrives grandesComo FAT32 pero sin el límite de 4 GB.
ext4LinuxPor defecto en Ubuntu y la mayoría de distros.
Btrfs / XFSLinuxAvanzados (instantáneas, gran tamaño).
swapLinuxNo es para datos: es la memoria virtual de Linux.
Trampas frecuentes
  • FAT32 no permite archivos > 4 GB.
  • swap en Linux ≠ datos. Es solo memoria virtual.
  • Si el firmware es UEFI, normalmente el disco es GPT. Si es BIOS, suele ser MBR.

Instalación de Windows y Linux
▼

Pasos comunes a cualquier instalación

  1. Descargar la imagen ISO oficial.
  2. Crear un medio de instalación (USB con Rufus / balenaEtcher) o cargar la ISO en una máquina virtual.
  3. Configurar el orden de arranque en la UEFI para arrancar desde el USB / DVD / ISO.
  4. Seguir el asistente: idioma, teclado, particionado, usuario, contraseña.
  5. Reiniciar y entrar en el SO recién instalado.
  6. Post-instalación: drivers, actualizaciones, programas básicos.

Instalación de Windows 11

  • Tipo de instalación: Personalizada (avanzada) para una instalación limpia.
  • Windows crea solas las particiones del sistema (EFI, MSR, datos).
  • Si no tienes clave de producto, selecciona «No tengo clave»: se instala igualmente.
  • Tras instalar: Windows Update, instalar drivers oficiales, navegador, antivirus si procede.

Instalación de Ubuntu

  • Modo «Probar Ubuntu»: arranca sin instalar (live USB).
  • Para instalar, opciones de particionado:
    • Borrar disco e instalar Ubuntu: lo más sencillo.
    • Instalar junto a Windows: arranque dual.
    • Más opciones: particionado manual.
  • Crea usuario y contraseña.
  • Tras instalar: sudo apt update && sudo apt upgrade, drivers privativos si los necesitas, software extra desde Software de Ubuntu o con apt install.
Recuerda

En el examen suele preguntar el tipo de instalación (Personalizada en Windows; Borrar disco en Ubuntu) y el gestor de paquetes de Ubuntu (apt).

Clonación, replicación y actualizaciones
▼

Clonación de discos

Clonar = hacer una copia exacta de un disco (o partición) en otro disco o en un archivo de imagen. Sirve para:

  • Pasar de HDD a SSD sin reinstalar nada.
  • Replicar el mismo SO en varios equipos del aula o la empresa.
  • Hacer una copia de seguridad completa antes de tocar el sistema.

Herramientas habituales

HerramientaPlataformaNotas
ClonezillaLive USBEstándar libre, en modo texto.
RescuezillaLive USBMisma idea, con interfaz gráfica.
Macrium ReflectWindowsComercial, fácil de usar.
ddLinuxComando de bajo nivel, potente y peligroso.

Despliegue masivo: imagen maestra

«Golden image»

Es una imagen de disco preconfigurada (con SO, drivers y programas instalados) que se replica a varios equipos. Acelera mucho el despliegue de un parque de PCs en una empresa o aula.

Sysprep (Windows)

sysprep /generalize deja una instalación de Windows lista para clonar: borra identificadores únicos (SID, nombre del equipo) para que cada copia se considere un PC nuevo al arrancar. Imprescindible antes de clonar Windows a varios equipos.

Actualizaciones

  • Windows Update: actualizaciones de seguridad y de funcionalidades. Se aplican automáticamente, suele requerir reinicio.
  • Linux: sudo apt update (refresca lista) + sudo apt upgrade (instala). Sin reinicio salvo si actualizas el kernel.
  • WSUS (Windows Server Update Services): centraliza actualizaciones en una empresa para no saturar la red.
★
Consejos para el examen
▼
Antes del examen

El examen es tipo test, 20 preguntas, 4 opciones, una sola correcta. No restan los fallos. Por tanto: no dejes nada en blanco.

  1. Diferencia siempre BIOS/MBR de UEFI/GPT. Esa pareja siempre cae.
  2. Windows 11 ≠ Windows 10: exige TPM 2.0, Secure Boot y procesador 64 bits.
  3. Comandos: dir (Windows) vs ls (Linux). Gestor de paquetes Ubuntu = apt.
  4. FAT32 no admite archivos > 4 GB. NTFS es el de Windows. ext4 el de Linux.
  5. swap en Linux = memoria virtual, no datos.
  6. Clonación: Clonezilla / Rescuezilla. Sysprep antes de clonar Windows.
  7. Lee las opciones enteras antes de marcar. Suelen tener distractores muy parecidos.
