Practica 26: Permisos y cifrado de archivos en Windows y Linux

Modulo: MF0219_2 - Instalacion y Configuracion de Sistemas Operativos
Unidad Formativa: UF0853 - Explotacion de las funcionalidades del sistema
Sesion: S4 - Permisos, cuotas y cifrado
Duracion estimada: 4 horas (1 sesion)
Modalidad: Trabajo individual
Herramientas: VM Windows 10 (Practica 06) + VM Ubuntu (Practica 07)
Nota: Esta practica requiere las maquinas virtuales de Windows 10 y Ubuntu de las practicas 06 y 07. Trabajaras con los dos sistemas operativos en paralelo para comparar como gestionan los permisos.

Objetivos de la practica

  • Entender el modelo de permisos NTFS de Windows y aplicarlo a archivos y carpetas.
  • Entender los permisos POSIX de Linux (rwx) y manejar chmod y chown.
  • Comparar como Windows y Linux protegen los datos.
  • Cifrar una carpeta o archivo con BitLocker / EFS en Windows y con un volumen cifrado en Linux.

Parte 1 - Permisos NTFS en Windows (60 min)

1.1 - Crear estructura de prueba

  1. Inicia la VM de Windows 10.
  2. Crea en el escritorio una carpeta llamada Permisos_Test.
  3. Dentro, crea dos archivos de texto: publico.txt y privado.txt.
  4. Escribe en cada uno una linea cualquiera para que tengan contenido.

1.2 - Crear un usuario adicional

  1. Abre Configuracion → Cuentas → Familia y otros usuarios.
  2. Pulsa en Agregar otro usuario en este equipo.
  3. Crea una cuenta local llamada invitado_test sin cuenta de Microsoft.
  4. Asigna una contrasena facil de recordar (la usaras solo para esta practica).

1.3 - Modificar permisos NTFS

  1. Haz clic derecho en privado.txt → Propiedades → pestana Seguridad.
  2. Pulsa Editar... y luego Agregar....
  3. Anade el usuario invitado_test y dale solo permiso de Lectura.
  4. En el archivo publico.txt, ya hay permisos para Usuarios. Comprueba que tienen Lectura y ejecucion.
  5. En el archivo privado.txt, denegar la escritura al grupo Usuarios.

1.4 - Comprobar el efecto

  1. Cierra la sesion del usuario actual.
  2. Inicia sesion con invitado_test.

Parte 2 - Permisos POSIX en Linux (60 min)

2.1 - Preparar el entorno

  1. Inicia la VM de Ubuntu.
  2. Abre la terminal (Ctrl+Alt+T).
  3. Ejecuta los siguientes comandos para crear archivos de prueba:
    mkdir ~/permisos_test && cd ~/permisos_test
    echo "Hola" > publico.txt
    echo "Secreto" > privado.txt
    ls -la

2.2 - Leer la cadena de permisos

Pega abajo la salida de ls -la y descompon los permisos del archivo privado.txt:

2.3 - Modificar permisos con chmod

Vamos a cambiar los permisos del archivo privado.txt para que solo el propietario pueda leerlo:

  1. chmod 600 privado.txt
  2. ls -la privado.txt → comprueba que ahora pone -rw-------.
  3. Anota tambien el equivalente con notacion simbolica: chmod o-r,g-r privado.txt.

2.4 - Cambiar el propietario con chown

  1. Crea un usuario adicional: sudo adduser pruebas (sigue las instrucciones).
  2. Cambia el propietario del archivo: sudo chown pruebas:pruebas privado.txt.
  3. Comprueba con ls -la.
  4. Como tu usuario, intenta leer el archivo: cat privado.txt. ¿Que ocurre?

Parte 3 - Cifrado de archivos (60 min)

3.1 - Cifrar un archivo en Windows con EFS

Aviso: EFS solo esta disponible en Windows Pro/Enterprise (no Home). Si tu Windows es Home, salta esta parte y usa la alternativa con 7-Zip cifrado.
  1. Sobre publico.txt, clic derecho → Propiedades → Avanzados....
  2. Marca Cifrar contenido para proteger datos. Acepta y aplica.
  3. Comprueba que el icono del archivo aparece con un candado.
  4. Cierra sesion e inicia con invitado_test. Intenta abrir el archivo. ¿Que pasa?

3.3 - Cifrar con 7-Zip (Windows Home)

  1. Instala 7-Zip si no lo tienes (7-zip.org).
  2. Clic derecho sobre privado.txt → 7-Zip → Anadir al archivo....
  3. En la ventana, formato 7z, escribe una contrasena, marca Cifrar nombres de archivo.
  4. Pulsa Aceptar y comprueba que el .7z resultante pide contrasena al abrirlo.

3.4 - Cifrar en Linux con gpg

  1. En la terminal de Ubuntu: echo "Datos secretos" > secreto.txt.
  2. Cifra con contrasena: gpg -c secreto.txt (te pedira una passphrase dos veces).
  3. Comprueba que se ha creado secreto.txt.gpg: ls -la.
  4. Borra el original: shred -u secreto.txt.
  5. Para descifrar: gpg secreto.txt.gpg.
