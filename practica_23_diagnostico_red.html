Practica 23: Diagnostico de Red y Seguridad Basica

Modulo: MF0219_2 — Instalacion y Configuracion de Sistemas Operativos
Unidad Formativa: UF0853 — Explotacion de las funcionalidades del sistema microinformatico
Bloque: Comunicacion por red y comparticion de recursos
Duracion estimada: 5 horas (1 sesion)
Modalidad: Trabajo individual / parejas
Herramientas: Oracle VirtualBox + VMs + Nmap + Wireshark
Preparacion previa: Antes de la sesion, descarga e instala Nmap (nmap.org) y Wireshark (wireshark.org) en tu equipo anfitrion o en la VM con Windows. Ambas herramientas son gratuitas. Tenerlas instaladas de antemano te permitira aprovechar al maximo el tiempo de la practica.
Escenario: Acabas de incorporarte como tecnico informatico en una pequena empresa. Tu responsable te pide que realices una auditoria basica de la red local: descubrir los dispositivos conectados, comprobar que puertos tienen abiertos, verificar que las reglas del firewall son correctas y capturar trafico de red para detectar posibles anomalias. Deberas documentar todo el proceso y tus hallazgos.

Objetivos de la practica

  • Utilizar netstat y ss para visualizar conexiones activas y puertos en escucha.
  • Trazar rutas de red con tracert (Windows) y traceroute (Linux) para analizar el camino de los paquetes.
  • Escanear redes y descubrir hosts y servicios con Nmap.
  • Configurar reglas de firewall en Windows Defender Firewall y ufw (Linux).
  • Realizar capturas basicas de trafico de red con Wireshark y aplicar filtros de analisis.

Parte 1 — Conexiones activas (30 min)

Vamos a examinar las conexiones de red activas y los puertos en escucha en ambos sistemas operativos.

1.1 Conexiones en Windows con netstat

  1. Abre una terminal de comandos como administrador en la VM con Windows.
  2. Ejecuta netstat -an para ver todas las conexiones y puertos en escucha.
  3. Ejecuta netstat -ano para ver ademas el PID del proceso asociado a cada conexion.
  4. Ejecuta netstat -ab para ver el nombre del programa asociado (requiere administrador).
  5. Identifica las conexiones con estado ESTABLISHED (activas) y LISTENING (en escucha).
C:\> netstat -an
Proto  Local Address          Foreign Address        State
TCP    0.0.0.0:135            0.0.0.0:0              LISTENING
TCP    0.0.0.0:445            0.0.0.0:0              LISTENING
TCP    192.168.1.10:49712     142.250.185.14:443     ESTABLISHED
TCP    192.168.1.10:50234     20.189.173.1:443       ESTABLISHED

C:\> netstat -ano
Proto  Local Address          Foreign Address        State           PID
TCP    0.0.0.0:135            0.0.0.0:0              LISTENING       1012
TCP    192.168.1.10:49712     142.250.185.14:443     ESTABLISHED     5840

1.2 Conexiones en Linux con ss

  1. Abre un terminal en la VM con Ubuntu.
  2. Ejecuta ss -tuln para ver puertos TCP y UDP en escucha.
  3. Ejecuta ss -tunap para ver todas las conexiones con el proceso asociado.
  4. Compara los resultados con lo obtenido en Windows.

Parte 2 — Trazar rutas de red (20 min)

Utilizaremos tracert (Windows) y traceroute (Linux) para ver el camino que siguen los paquetes hasta llegar a su destino.

2.1 Trazar rutas desde Windows

Parte 3 — Escaneo de red con Nmap (60 min)

Nmap (Network Mapper) es una herramienta gratuita y de codigo abierto para descubrimiento de red y auditoria de seguridad. Descargala desde nmap.org.

IMPORTANTE — Etica y legalidad: Escanear redes y puertos sin autorizacion es ilegal en muchos paises. En esta practica SOLO escanearemos nuestra propia red local y nuestras maquinas virtuales. Nunca escanees redes ajenas, servidores de Internet u ordenadores de terceros sin permiso explicito por escrito.

3.1 Instalacion de Nmap

  1. Windows: Descarga el instalador desde nmap.org/download y ejecutalo. Incluye Zenmap (interfaz grafica).
  2. Linux: Ejecuta sudo apt install nmap -y en el terminal.
  3. Verifica la instalacion ejecutando nmap --version en ambos sistemas.

3.2 Descubrimiento de hosts en la red

  1. Primero, averigua tu IP y la subred con ipconfig (Windows) o ip a (Linux).
  2. Ejecuta nmap -sn 192.168.1.0/24 (ajusta la subred a tu caso) para descubrir hosts activos sin escanear puertos.
  3. Anota todos los dispositivos encontrados en la tabla.

Parte 4 — Firewall en Windows (40 min)

Vamos a trabajar con Windows Defender Firewall con Seguridad Avanzada para crear, probar y eliminar reglas de firewall.

4.1 Abrir la consola de Firewall avanzado

  1. Pulsa Win + R y escribe wf.msc, luego pulsa Enter.
  2. Se abrira la ventana de Windows Defender Firewall con Seguridad Avanzada.
  3. Observa las tres secciones principales: Reglas de entrada, Reglas de salida y Reglas de seguridad de conexion.
  4. Examina algunas de las reglas existentes para familiarizarte con la interfaz.

4.2 Crear regla para bloquear puerto 80 (HTTP)

  1. Haz clic en Reglas de entrada en el panel izquierdo.
  2. Haz clic en Nueva regla... en el panel derecho.
  3. Selecciona Puerto y haz clic en Siguiente.
  4. Selecciona TCP y escribe 80 en "Puertos locales especificos". Haz clic en Siguiente.
  5. Selecciona Bloquear la conexion. Haz clic en Siguiente.
  6. Deja marcados los tres perfiles (Dominio, Privado, Publico). Haz clic en Siguiente.
  7. Escribe un nombre descriptivo: Practica23 - Bloquear HTTP puerto 80. Haz clic en Finalizar.

4.3 Probar la regla

  1. Abre un navegador en la VM con Windows e intenta acceder a una pagina web HTTP (no HTTPS).
  2. Ejecuta nmap -p 80 [IP_Windows] desde la VM con Linux para verificar que el puerto aparece como filtered.
  3. Documenta el resultado.

4.4 Crear regla para permitir un programa especifico

  1. Haz clic en Reglas de entrada > Nueva regla...
  2. Selecciona Programa y haz clic en Siguiente.
  3. Selecciona Esta ruta de acceso del programa y busca un ejecutable (por ejemplo, C:\Windows\System32\mspaint.exe).
  4. Selecciona Permitir la conexion. Haz clic en Siguiente.
  5. Deja los tres perfiles marcados. Haz clic en Siguiente.
  6. Nombre: Practica23 - Permitir Paint. Haz clic en Finalizar.

4.5 Eliminar las reglas de prueba

  1. En Reglas de entrada, busca las reglas que creaste (busca "Practica23").
  2. Haz clic derecho sobre cada regla y selecciona Eliminar.
  3. Confirma la eliminacion.
  4. Verifica que las reglas ya no aparecen en la lista.

Parte 5 — Firewall en Linux con ufw (30 min)

ufw (Uncomplicated Firewall) es la herramienta de firewall por defecto en Ubuntu. Proporciona una interfaz sencilla para gestionar reglas de iptables.

5.1 Activar y comprobar ufw

  1. Abre un terminal en la VM con Ubuntu.
  2. Ejecuta sudo ufw enable para activar el firewall (confirma con "y").
  3. Ejecuta sudo ufw status verbose para ver el estado actual y la politica por defecto.
$ sudo ufw enable
Firewall is active and enabled on system startup

$ sudo ufw status verbose
Status: active
Logging: on (low)
Default: deny (incoming), allow (outgoing), disabled (routed)
New profiles: skip

5.2 Crear reglas

  1. Ejecuta sudo ufw allow 22/tcp para permitir conexiones SSH.
  2. Ejecuta sudo ufw deny 80/tcp para bloquear conexiones HTTP entrantes.
  3. Ejecuta sudo ufw allow 443/tcp para permitir conexiones HTTPS.
  4. Ejecuta sudo ufw status numbered para ver las reglas con numeros.
$ sudo ufw allow 22/tcp
Rule added

$ sudo ufw deny 80/tcp
Rule added

$ sudo ufw allow 443/tcp
Rule added

$ sudo ufw status numbered
Status: active
     To                         Action      From
     --                         ------      ----
[ 1] 22/tcp                     ALLOW IN    Anywhere
[ 2] 80/tcp                     DENY IN     Anywhere
[ 3] 443/tcp                    ALLOW IN    Anywhere

5.3 Probar las reglas

  1. Desde la VM con Windows, ejecuta nmap -p 22,80,443 [IP_Linux].
  2. Verifica que el puerto 22 aparece como open, el 80 como filtered/closed y el 443 depende de si hay servicio escuchando.
  3. Documenta los resultados en la tabla.

5.4 Eliminar una regla

  1. Ejecuta sudo ufw status numbered para ver los numeros de cada regla.
  2. Ejecuta sudo ufw delete 2 para eliminar la regla que bloquea el puerto 80 (confirma con "y").
  3. Verifica con sudo ufw status que la regla se ha eliminado.

Tabla de comandos ufw y resultados

Comando Accion esperada Resultado obtenido
sudo ufw enable Activar el firewall
sudo ufw allow 22/tcp Permitir SSH
sudo ufw deny 80/tcp Bloquear HTTP
sudo ufw allow 443/tcp Permitir HTTPS
sudo ufw status numbered Ver reglas numeradas

Parte 6 — Captura de trafico con Wireshark (40 min)

Wireshark es el analizador de protocolos de red mas utilizado del mundo. Es gratuito y se descarga desde wireshark.org.

6.1 Iniciar una captura

  1. Abre Wireshark en tu VM con Windows (o en la de Linux).
  2. Selecciona la interfaz de red activa (la que tiene trafico, normalmente "Ethernet" o "eth0").
  3. Haz doble clic en la interfaz o pulsa el boton de aleta de tiburon (Start) para iniciar la captura.
  4. Veras que empiezan a aparecer paquetes en tiempo real.

6.2 Generar trafico para capturar

  1. Abre un navegador web y visita varias paginas: google.es, wikipedia.org, example.com.
  2. Haz un ping a 8.8.8.8 desde el terminal.
  3. Deja la captura corriendo durante 1-2 minutos.
  4. Pulsa el boton de cuadrado rojo (Stop) para detener la captura.

6.3 Aplicar filtros de visualizacion

En la barra de filtros de Wireshark, prueba los siguientes filtros:

Filtro Que muestra N. de paquetes encontrados
dns Consultas y respuestas DNS
http Trafico HTTP (no cifrado)
tcp.port==443 Trafico HTTPS / TLS
icmp Paquetes ping (ICMP)
ip.addr==8.8.8.8 Todo el trafico con Google DNS

6.4 Analizar consultas DNS

  1. Aplica el filtro dns.
  2. Busca las consultas que corresponden a las paginas que visitaste.
  3. Haz clic en un paquete DNS y observa los detalles en el panel inferior.
  4. Identifica: dominio consultado, tipo de consulta (A, AAAA), IP resuelta.

Tabla de consultas DNS encontradas

Dominio consultado Tipo de consulta IP resuelta Servidor DNS usado
Nota sobre HTTPS: El trafico HTTPS esta cifrado con TLS, por lo que Wireshark no puede ver el contenido de las paginas que visitas. Solo podras ver el "handshake" TLS y los datos cifrados. Para ver trafico HTTP legible, necesitas una pagina que use HTTP sin cifrar (cada vez son menos comunes).
